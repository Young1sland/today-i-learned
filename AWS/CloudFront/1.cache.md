## CloudFront 캐싱

### 캐싱 티어
- CloudFront의 캐싱은 2티어
- 요청 순서
    1.  Edge Location  
    2.  Regional Edge Cache(Edge Location 상위의 캐시라고 보면 됨)  
    3.  Origin  

### Cache Key
- 요청에 따라 어떤 캐시 내용을 보여줄지를 결정하는 정보의 조합
    - 각 오브젝트는 고유의 Cache Key 단위로 캐시 됨
- Cache Hit: 뷰어가 특정 Cache Key로 오브젝트를 요청하였을 때, Edge Location에서 해당 Cache 오브젝트를 가지고  
  있어 원본에 요청 과정 없이 제공할 수 있는 상황
  - Origin의 부하 경감 가능
  - 더 빠르게 컨텐츠 제공 가능
  - CDN은 Cache Hit이 많을수록 더 좋은 퍼포먼스 제공 가능
- 주요 Cache Key 구성 요소
  - 기본: 경로
  - 추가 가능 요소 : query string, HTTP Header, Cookie
  - 예시
    - /img.png를 가져올 때 기본으로 url path가 Cache Key로 사용 됨. EdgeLocation에 있으면 Cache Hit
    - /img.png?color=blue로 요청했을 경우 url path만 Cache Key로 설정되어 있다면 그대로 기존 img.png를 EdgeLocation에서 가져옴.
    - Cache Key에 query string color를 추가해두면 /img.png?color=blue가져올 때 다른걸로 인식하여 Origin에 요청하게 됨
- HTTP Header baed Cache
  - HTTP Header를 Cache Key로 활용
  - 언어별 캐싱
  - 디바이스 타입별로 캐싱
    - CloudFront에서 별도로 User-Agent를 기반으로 전용 헤더를 생성
    - CloudFront-Is-Desktop-Viewer/CouldFront-Is-Mobile-Viewer/CloudFront-Is-SmartTV-Viewer 등
  - 지역별 캐싱
    - CloudFront에서 전용 헤더 생성 CloudFront-Viewer-Country
  - 헤더명은 대소문자를 구분하지 않지만, 값은 구분
- Cookie based Cache
  - Cookie 활용하지 않는 HTTP 서버 혹은 S3에서 사용할 경우 퍼포먼스 저하될 수 있음
### Cache 만료
- 캐시된 Object는 일정기간(TTL, Time to Live) 이후 만료
- 캐싱 만료 후 다음 요청이 올 경우 CloudFront는 Origin에 Object 갱신 여부 확인
- Origin이 304 Not Modified를 줄 경우 갱신 없이 그대로 Edge Location 내용 넘겨줌
- 200 OK와 파일을 줄 경우, 갱신
### Cache TTL
- TTL(Time To Live): Cache Object를 얼마나 오래 보관할지에 관한 설정
  - 기본 24시간, 모든 CloudFront의 Object에 적용
  - 파일 단위에서는 Origin에서 Cache-Control 헤더 혹은 Expires 헤더를 포함하면 EdgeLocation에서 이 파일에만 해당 사항 적용해 조절 가능
- TTL의 종류
  - Default TTL: 별도의 설정 없을 경우 부여되는 기본 TTL
  - Minimum TTL: 최소 TTL, 즉 파일 단위 컨트롤에서 줄 수 있는 최소 TTL
  - Maximum TTL: 최대 TTL, 즉 파일 단위 컨트롤에서 줄 수 있는 최대 TTL
  - Origin에서 Cache-Control,Exires헤더에 이 Min,Max범위를 넘어가는 값을 설정하면 무시 됨
- Cache TTL 컨트롤
  - 파일 단위에서는 Origin에서 Cache-Control헤더 혹은 Expires 헤더를 포함해서 조절 가능
  - Cache-Control헤더: 얼마나 오래 Object를 Cache하는지 기간 설정
    - max-age: CloudFront와 브라우저 둘 다 영향
    - s-maxage: CloudFront만 영행
    - no-cache, no-store: 캐싱하지 않음(단, Min TTL이 0이상일 경우 Min TTL로 최저 설정)
  - Expires: Cache가 만료되는 정확한 시각을 설정
    - CloudFront와 브라우저 영향
- CloudFront의 Min/Max TTL 범위 안에서만 설정 가능
- 예시: Origin 응답 시 Cache-Control: max-age=3600을 헤더에 넣어 EdgeLocation이 넘겨 받으면 이에 따라 3600초 이후 만료 됨.

### Cache Policy
- 캐싱 관련된 내용을 정책으로 정의하여 CloudFront에 적용 가능. 만들어두면 두고두고 사용 가능..
- 주요 설정
  - 어떤 키(HTTP Header, Cookie, querystring 등)으로 컨텐츠를 캐시하는지
  - 얼마나 오래 캐시하는지(TTL), 컨텐츠를 압축 저장 관련 설정
- 두가지 종류
  - Managed: AWS에서 직접 생성한 Policy
  - Custom: 직접 설정
