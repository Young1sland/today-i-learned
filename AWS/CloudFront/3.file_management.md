## CloudFront의 파일관리
### 2가지 파일 관리 방법
- 싱글파일: 파일명을 유지한 체로 캐시 만료 처리, 파일변경 등을 처리
  - 별도로 클라이언트(뷰어) 측 업데이트 필요 없음
  - 캐시 만료 전 EdgeLocation의 파일을 업데이트 하려면 Invalidation 필요.
  - Invalidation은 강제로 만료시킴을 뜻함.
- 버저닝: 파일 이름에 다양한 방법으로 버전을 두어서 관리
  - 별도로 Invalidation 필요 없음
  - 파일 변경 필요 시 클라이언트(뷰어) 측 업데이트 필요
- 예시
  - 싱글파일: 뷰어에서 img.png를 요청해 사용하다 변경이 필요하다. 이 경우 Edge Location의 img.png를 invalidation시켜야 한다.이 경우 뷰어 측에서는 수정이 없다.
  - 버저닝: 파일명에 img_1.png과 같이 버젼식별을 붙인다. 이후 파일 변경이 필요하면 img_2.png로 요청한다.


### Invalidation
- 캐시 만료 전 파일 갱신
- 버저닝 아닌 경우 캐시 만료 전 새 파일 제공원할 경우 invalidation 필요
- 경로 기반으로 invalidation할 수 있다.
  - /img/img1.png
  - /img/*
  - /img/img*
- 한번에 최대 3000파일까지 Invalidate 가능
- 한달에 1000 path Invalidation은 무료(계정 전체 Distribution 통합), 이후 한번 경로당 $0.005
