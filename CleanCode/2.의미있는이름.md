### 의도를 분명히 밝혀라
변수(혹은 함수나 클래스)의 존재 이유, 수행 기능, 사용방법에 대한 의도가 분명하게 이름을 지어야 함.
```java
int elapsedTimeInDays;
int daysSinceCreation;
int daysSinceModification;
int fileAgeInDays;
```

### 그릇된 정보를 피하라
코드에 그릇된 단서를 남겨서는 안된다. 널리 쓰이는 의미가 있는 단어를 다른 의미로 사용해서는 안 된다.
- 계정을 그룹으로 묶을 때 accountList라 명명할 경우 실제 List가 아니라면 프로그래머에게 그릇된 정보를 제공하는 셈이다. accountGroup, bunchOfAccounts, Accounts가 좋겠다.
- 이름으로 그릇된 정보를 제공하지 않도록 한다. 소문자 L, 대문자 O는 각각 1,0과 유사하므로 변수로 사용하지 않도록 하자.

### 의미 있게 구분하라
이름이 달라야 한다면 의미도 달라져야 한다. noise word를 추가하여 이름을 구별하는 방식은 적절하지 못하다.
```java
public static void copyChars(char a1[], char a2[]){
    for(int i=0; i< a1.length; i++>){
        a2[i] = a1[i];
    }
}
//a1,a2를 source, destication으로 명명했다면 코드 읽기가 훨씬 쉬워진다.

//Data,Info,a,an,the는 의미가 불명명한 noise word다. 의미를 분명히 다르게 정의했다면 사용해도 무방하다. ex. a는 지역변수, the는 함수에만 붙임
getActiveAccount();
getActiveAccountInfo(); 
```
