### HpyerLogLog(HLL)

확률적 데이터 구조(Probabilistic data structure)로 set의 대략적인 cardinality(멤버수)를 측정하는 것에 사용. 이름은 log를 두번 쓰는 것에서 유래. log(log(list))

- Fixed-Low Memory footprint: set의 멤버수와 상관없이 12KB의 적인 메모리 사용.
- Trade-off between memory usage and accuray: 메모리를 적게 사용하지만 정확도는 좀 떨어질 수 있다. (Error rate is 0.81%, 992 ~ 1008 out of 1000)
- Use Case
  - unique vists to a website, unique ip address,unique items in a shoppping cart, uique count with good security
  - HLL은 멤버의 String을 hash function으로 0,1로 이뤄진 binary representation으로 변환하여 저장한다.
  - 예를 들면 방문한 적이 있는 사용자인지 체크하기 위해 redis의 set의 방문자를 계속 더해가고 set에 존재하지 않을 경우 첫방문으로 처리하는 경우, 사용자가 많을 경우 메모리를 많이 사용하게 된다. 이 때 HLL을 사용하면 유용하다.
- Commands

  ```py

    # PFADD: HLL SET에 맴버 추가. 새로운 set이면 1, 아니면 0리턴
    print(r.pfadd('unique_visitors', 'visitor1', 'visitor2', 'visitor3', 'visitor1'))
    # 1

    # PFCOUNT: 멤버수를 조회. (대략적인 값)
    approx_unique_visitors = r.pfcount('unique_visitors')
    # 3

    r.pfadd('hll1', 'a', 'b', 'c', 'd', 'e')
    r.pfadd('hll2', 'c', 'd', 'e', 'f', 'g')

    # PFMERGE: 두 set을 Union한다.
    r.pfmerge('hll_union', 'hll1', 'hll2')
    approx_union_count = r.pfcount('hll_union')
    # 7
  ```

### List

Redis List는 삽입한 순서가 유지는 sequences이며 Double Linked List로 구현되어 있다. head,tail모두에 삽입 가능하며 index로 접근가능하다.

```py
# list_ex.py

# LPUSH: 왼쪽에 추가, RPUSH: 오른쪽에 추가
r.lpush('mylist', 'element1')
r.rpush('mylist', 'element2')

# LINDEX: index로 elements 정보 가져오기
element_at_index_0 = r.lindex('mylist', 0)

# LLEN: list의 길이 구하기
list_length = r.llen('mylist')

# LPOP: 왼쪽에서 POP 함. RPOP: 오른쪽에서 POP 함.
popped_element_left = r.lpop('mylist')
popped_element_right = r.rpop('mylist')

# LPUSH로 여러 elements 추가하기 
r.lpush('mylist', 'element3', 'element4', 'element5', 'element4')

# LRANGE: index로 범위의 elements 구하기
elements_range = r.lrange('mylist', 0, -1)

# Add some elements to the list
r.rpush('fruits', 'apple', 'banana', 'apple', 'apple', 'banana', 'orange', 'apple')

# LPOS rank 옵션: 키워드로 검색하여 index를 구하되 같은 elements가 존재할 경우 rank 즉 N번째 해당하는 position을 구한다.
# Find the 2-based position of 'apple' in the list
position_rank_2 = r.lpos('fruits', 'apple', rank=2)
# 2

# LPOS maxlen 옵션: 키워드로 검색하여 index를 구하되 같은 list의 maxlen 이하까지만 찾음
# Find the position of 'banana' in the list with MAXLEN=4
position_maxlen = r.lpos('fruits', 'banana', maxlen=4)
# 1

# LPOS count 옵션: 키워드로 검색하여 index를 구하되 count개만 구한다.
position_count = r.lpos('fruits', 'apple', count=2)
# [0,2]

# LSET: index에 값을 넣는다.(변경)
r.lset('fruits', 2, 'pineapple')

# LTRIM: index 사이의 값으로 짤라 해당 값들만 가진다.
r.ltrim('fruits', 1, 2)
# ['banana', 'pineapple']

# LINSERT before: 특정값 앞에 삽입
result_before = r.linsert('fruits', 'BEFORE', 'pineapple', 'dragonfruit')

# LINSERT after: 특정값 뒤에 삽입
result_after = r.linsert('fruits', 'AFTER', 'pineapple', 'watermelon')

# LREM: 특정 keyward의 element를 삭제한다. 삭제된 수를 리턴한다.
removed_count = r.lrem('fruits', 0, 'watermelon')
# 1
```

### Lua Language

Redis Server에서 Lua scripts를 사용할 수 있다. 복잡한 로직을 사용할 때 atomically 동작하게 해준다.
(Atomicity: 여러 연산이 single transaction에 실행되고 에러 발생 시 모두 무효)

- Use Cases
  - Atomic Operations
  - Reduced Network Round-Trips
  - Complex Logic
  - Server-Side Processing: no client processing
  - Performance Optimization: Minimize data transfer
  - Cache Invalidation And Eviction
- 문법

    ```lua
    print("Hello Lua")
    local value = 10+1

    if value > 0 then
        print("value is positive")
    end

    if value ~=0 then
        print("value is non zero")
    end

    if value==0 then
        print("value is 11")
    end

    -- Array
    local country = {"South Korea", "United States", "Japan"}

    -- Lua는 첫번째 index가 1
    print(country[1]) -- South Korea

    -- length function #
    print(#country) -- 3

    -- Array에 삽입하기
    table.insert(country,'China') -- 4번째에 들어감

    -- Array 출력
    for i,v in ipairs(country) do
        print(i,v)
    end
    -- 1 South Korea
    -- 2 United States
    -- 3 Japan
    -- 4 China

    -- for 문
    for i=1,5 do
        print(i)
    end

    -- Dictionary
    local car = {color='white', name='tesla'}
    print(car['name'])

    -- Dictionary 출력
    for k,v in pairs(car) do
        print(k,v)
    end
    ```

- KEYS, ARGV는 lua의 예약어로 각각 key값, arguments를 가진다.
- eval호출 시 , key length, key와 ,argv를 넘겨준다.
- lua에서 redis.call로 실행하고 return해주면 eval 리턴값으로 받을 수 있ㄷ.

    ```py

    # Establish connection to Redis
    r = redis.Redis(host='redis', port=6379, db=0, decode_responses=True)

    # Your Lua script
    lua_script = """
    local key1 = KEYS[1]
    local key2 = KEYS[2]
    local increment = ARGV[1]
    redis.call('DEL', key1)
    redis.call('DEL', key2)
    redis.call('INCRBY', key1, increment)
    redis.call('INCRBY', key2, increment)

    return redis.call('MGET', key1, key2)
    """
    # The keys and arguments for the script
    keys = ['key1', 'key2']
    args = [5]

    # Execute the script
    result = r.eval(lua_script, len(keys), *keys, *args)

    # Print the result
    print(result)
    #['5','5']
    ```

### lock 통한 Concurrency problem 해결

SET NX를 활용해서 key가 존재하지 않을 때만 lock을 얻게 할 수 있다.

#### 1. SimleLock

- r.set nx 옵션으로 key가 없을 때만 True 리턴. key가 존재하는 경우 sleep(0.1) 후 계속 lock 얻기 위한 시도

   ```py
   # redis_with_simple_lock.py

   # Establish connection to Redis
   r = redis.Redis(host='redis', port=6379, db=0, decode_responses=True)

   def acquire_lock(lock_name, acquire_timeout=10):
       """Attempt to acquire a lock with the given name."""
       lock_key = f"lock:{lock_name}"
       while True:
           # Attempt to acquire the lock
           if r.set(lock_key, "locked", ex=acquire_timeout, nx=True):
               return True
           # Sleep for a short interval before retrying
           time.sleep(0.1)

   def release_lock(lock_name):
       """Release the lock with the given name."""
       lock_key = f"lock:{lock_name}"
       # Delete the lock key
       r.delete(lock_key)

   # Example usage:
   if acquire_lock("my_lock"):
       try:
           # Critical section - do some work here
           print("Lock acquired, performing critical section...")
           time.sleep(5)  # Simulate some work
       finally:
           print("Lock is released...")
           release_lock("my_lock")
   else:
       print("Failed to acquire lock, another process is holding the lock.")
   ```

#### 2. Distributed Locks (실제 사용 시)

우의 simple lock은 문제가 많으므로 실제 사용 시는 클러스터를 활용한 하기 방법을 사용한다.
<https://redis.io/docs/manual/patterns/distributed-locks/>

- 파이썬에서 redlock-py를 활용한 예제. Dockerfile에서 보듯 edlock-py 설치 필요

```py
# redis_with_redlock.py
# https://github.com/SPSCommerce/redlock-py
class RedlockHelper(object):
    def __init__(self):
        # Create a RedLock instance with connection details to your Redis servers
        # You can specify multiple Redis servers for fault tolerance
        servers = [
            {"host": "redis", "port": 6379, "db": 0}
        ]
        self.dlm = Redlock(servers)
        self.my_lock = None

    def acquire_lock(self, lock_name, ttl=5000):
        """Attempt to acquire a lock with the given name."""
        lock_key = f"lock:{lock_name}"
        while True:
            self.my_lock = self.dlm.lock(lock_key, ttl)
            if self.my_lock:
                return True
            # Sleep for a short interval before retrying
            time.sleep(0.1)

    def release_lock(self):
        """Release the lock with the given name."""
        # Delete the lock key
        self.dlm.unlock(self.my_lock)

if __name__ == "__main__":
    redlockHelper = RedlockHelper()
    if redlockHelper.acquire_lock("my_lock"):
        try:
            # Critical section - do some work here
            print("Lock acquired, performing critical section...")
            time.sleep(5)  # Simulate some work
        finally:
            print("Lock is released...")
            redlockHelper.release_lock()
    else:
        print("Failed to acquire lock, another process is holding the lock.")
```

### Publish & Subscribe

- Publishers: Redis channel에 message를 보내는 clients
- Subscribers: Redis Channel로 부터 message를 받는 clients
- Channels: 브로커. names를 식별되며 Clients는 여러 채널을 동시에 구독가능하다.
- Message: data units
- Pattern Subscription: string 패턴으로 match되는 여러 채널을 구독하고 해지 가능(psubscribe, wildcard 사용)
- Messagge Delivery: one-to-many로 동작.

#### Publisher

```py

# Connect to Redis server
r = redis.Redis(host='redis', port=6379, db=0, decode_responses=True)

while True:

    if message.lower() == 'exit':
        break
    r.publish('my_channel', message)

pubsub.close()
```

### Streams

 일련의 메시지 스트림을 저장하고 처리하는 데 사용.

- pubSub과 기본적으로 비슷. Streams라는 데이터 타입 안에 프로듀서가 메시지를 계속 남기면 consumer가 이거 읽어 처리함.
- Ordered Message Stream: 모든 메시지가 ordered sequence를 유지
- Consumer Groups: 여러 consumer가 동시에 메시지를 읽을 수 있음
- Acknowledment and Message Delivery: ACK로 message가 처리되어는지 확인. consumer가 메시지를 어디까지 읽었는지, 어떤 컨슈머가 어디까지 읽었는지 status를 저장할 수 있다.
- Consumer State Tracking: 컨슈머 그룹의 각 consumer의 상태를 기록.
- Stream Trim and Garbage Collection: expired 메시지를 자동 삭제하거나 streams size를 control할 수 있다.
- Use Cases: messaging systems, activity feeds, log processing pipelines

#### 심플한 예시

streams에 메시지를 넣고 단순하게 읽어 프린트한다. streams에서 message ID와 함께 message를 읽어올 수 있다. message ID는 지정하지 않으면 unix epoch으로 자동할당된다.

```py
# streams.py
# Establish connection to Redis
r = redis.Redis(host='redis', port=6379, db=0, decode_responses=True)

stream_key = 'mystream'
r.delete(stream_key)

# Add messages to a stream
messages = [
    {'field1': 'value1', 'field2': 'value2'},
    {'field1': 'value3', 'field2': 'value4'},
    {'field1': 'value5', 'field2': 'value6'}
]
for message in messages:
    r.xadd(stream_key, message) # type: ignore

# Read messages from the stream
# Start reading from the beginning (min='-' first, max='+' last)
stream_range = r.xrange(stream_key, min='-', max='+') # type: ignore
for message_id, fields in stream_range: # type: ignore
    print(f"Message ID: {message_id}, Fields: {fields}")
```

messages = [

#### 예시 : Consumer Group 생성

```py
# Establish connection to Redis
r = redis.Redis(host='redis', port=6379, db=0, decode_responses=True)

# Stream key and group name
stream_key = 'mystream'
group_name = 'mygroup'

# Add the first message with the custom message ID
first_message = {'field1': 'value1', 'field2': 'value2'}
r.xadd(stream_key, first_message) # type: ignore

# Get information about consumer groups associated with the stream
consumer_groups_info = r.execute_command('XINFO', 'GROUPS', stream_key)

# Check if the specified consumer group exists
group_exists = any(group[1] == group_name for group in consumer_groups_info)
if group_exists:
    print(f"Consumer group '{group_name}' exists.")
else:
    print(f"Consumer group '{group_name}' does not exist.")
    # Create a new consumer group
    created = r.xgroup_create(stream_key, group_name, id='0', mkstream=True)
    print(f"Consumer group '{group_name}' created: {created}")

# Add the first message with the custom message ID
first_message = {'field1': 'value1', 'field2': 'value2'}
r.xadd(stream_key, first_message) # type: ignore

consumer_name = 'consumer1'

# Read messages from the stream within the consumer group
stream_entries = r.xreadgroup(group_name, consumer_name, {stream_key: '>'}, count=10) # '>' ack한 다음 이후부터 읽겠다
for stream, messages in stream_entries: # type: ignore
    print(f"Stream: {stream}")
    for message_id, fields in messages:
        print(f"Message ID: {message_id}, Fields: {fields}")
        acknowledged = r.xack(stream_key, group_name, message_id)
        print(f"Message '{message_id}' acknowledged: {acknowledged}")

first_message = {'field1': 'value1', 'field2': 'value2'}
r.xadd(stream_key, first_message) # type: ignore

# Read messages from the stream within the consumer group
stream_entries = r.xreadgroup(group_name, consumer_name, {stream_key: '>'}, count=10)
for stream, messages in stream_entries: # type: ignore
    print(f"=> After Acked Stream: {stream}")
    for message_id, fields in messages:
        print(f"Message ID: {message_id}, Fields: {fields}")


r.xgroup_destroy(stream_key, group_name)
r.delete(stream_key)

```
