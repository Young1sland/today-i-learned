1. 인메모리 DB의 필요성
- 디스크 대신 메모리에 저장함으로써 I/O 성능을 개선
- 메모리 특성상 영구성이 보장되지 않으므로 적절한 백업 시스템 구축 필요
- Redis, MemCached가 사용되고 있으며 더 다양한 기능 제공되는 redis를 더 많이 사용

2. Redis 서버 구축 고려사항
    1) 데이터 관리
       1) 캐싱 전략
             - Cache Aside : App에서 캐시에 데이터 요청. 없으면 DB에 요청하고 읽어온 데이터를 캐시에 저장 함.
             - Read Through : App에서 캐시에 데이터 요청.없으면 DB에 요청하고 캐시에 저장한 뒤 App에 응답을 보냄
             - Write Through : App에서 캐시에 데이터 쓰고 DB에 데이터를 씀
             - Write Back : 캐시에 데이터 쓰고 약간의 지연 후 한번에 DB에 데이터 씀
       2) 데이터 백업
             - 데이터 휘발성 보완 위해 MemCached와 달리 Redis는 RDB, AOF 백업 옵션 지원 함. 데이터 중요도 높다면 백업 옵션 고려
             - RDB (Redis Database) : 특정 시점에 메모리에 저장된 젖체 데이터를 바이너리 파일로 저장
             - AOF(Append Only File) : 데이터 변경이 있을 때마다 별도의 팡딜에 해당 내용 추가
    2) 모드 선택    
       
       ||스탠드얼론|센터널|클러스터|
       |---|:---:|:---:|:---:|
       |노드구성|마스터1개|마스터1개 슬레이브 N개, 센티널 3개|마스터 N개, 슬레이브 N개|
       |장애복구|RDB 또는 AOF 파일을 수동으로 재시작| 센티널이 감지하여 슬레이브를 마스터로 승격|노드가 서로 감지하여 장애 마스터를 슬레이브로 승격|
       |가용성|X|O|O|
       |이중화|X|O|O|
       |샤딩|X|X|O|
