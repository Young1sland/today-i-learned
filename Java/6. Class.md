# Class

### Class 선언
하나의 소스파일에 복수 개의 class를 선언할 수 있다. 그러나 public class는 소스 파일명과 동일한 class만 선언가능하다.

### 객체 생성과 Class 변수
new 연산자는 객체를 생성시킨 후 객체의 주소를 리턴한다.

### Class 구성 멤버
Constructor, Field, Method

### Constructor 선언과 호출
public class로 선언되면 기본 생성자도 Public이 붙고 private class면 생성자도 private이다.

### 필드 초기화
객체마다 동일한 값을 갖고 있다면 필드 선언 시 초기값을 대입하는 것이 좋고, 객체마다 다른 값을 가져야 한다면 생성자에서 필드를 초기화하는 것이 좋다.

### 다른 Constructor 호출하여 중복 코드 줄이기
생성자 오버로딩이 많아질 경우 this(...)를 사용하여 공통 코드를 가지고 있는 생성자를 호출하는 방법으로 개선할 수 있다.
```java
public class Car {
  String model,color;
  int maxSpeed;
  Car(String model){
    this(model, "Siver", 250);
  }
  Car(String model, String color){
    this(model,color,250);
  }
  Car(String model, String color, int maxSpeed){
    this.model = model;
    this.color = color;
    this.maxSpeed = maxSpeed;
  }
  
}
```

### 가변길이 매개 변수
매개값들은 자동으로 배열 항목으로 변환되어 메소드에서 사용 됨. 메소드 호출 시 직접 배열을 매개값으로 제공해도 됨
```java
public class Computer {
  int sum(int ... values){
    int sum = 0;
    for(int i=0;i<values.length; i++){
      sum+= values[i];
    }
    return sum;
  }
}

Computer com = new Computer();
int result = com.sum(1,2,3);
System.out.println(result); //6
int result2 = com.sum(new int[]{1,2,3,4,5}); //배열을 인자로 넣기
System.out.println(result2); //15
```

### 인스턴스 멤버
- 인스턴스 필드는 객체마다 따로 존재하지만 인스턴스 메소드는 메소드 영역에 저장되고 공유된다. 
- 메소드는 코드의 덩어리이므로 객체마다 저장한다면 중복저장으로 인해 메모리 효율이 떨어짐.
- 메소드 코드는 메소드 영역에 두고 공유해서 사용하고 객체 없이는 사용하지 못하도록 제한을 걸어 둠.
```java
public class Car {
  //인스턴스 필드
  int val;

  //인스턴스 메소드 
  void setVal(int val){}
}
```

### 정적 멤버
- 클래스 로더를 이용해서 클래스를 메소드 영역 저장하고 사용
- 정적 멤버는 메소드 영역의 클래스에 고정적으로 위치. 객체 생성 없이 클래스를 통해 바로 사용 가능
- 공용적인 필드는 정적 필드로 선언, 인스턴스 필드를 이용하지 않는 메소드는 정적 메소드로 선언하는 것이 좋음
- 복잡한 초기화 작업이 필요한 경우 정적 블록을 이용하자. 정적 블록은 클래스가 메모리로 로딩될 때 자동 실행 됨.
- 정적 메소드나 정적 블록에서 인스턴스 멤버를 사용하고 싶다면 객체를 먼저 생성하고 참조 변수로 접근해야 함
```java
//정적 블록
public class StaticBlock {
  static String A = "A";
  static String B = "B";
  static String info;

  static {
    info = A+"_"+B;
  }
}
//System.out.println(StaticBlock.info);
```

### final 필드와 상수
- final 필드는 선언 시나 생성자에서 초기값을 대입해 줘야 함
- 상수는 static final로 선언하고 선언시에 초기값을 주거나 정적 블록에서 초기화 가능하다.
- 상수 이름은 모두 대분자로 작성하는 것이 관례이며 여러 단어를 혼합해야 하는 경우 '_'로 연결한다.
```java
public class ConstClass {
  static final double LENGTH = 6400;
  static final double AREA_A;

  static {
    AREA_A = LENGTH * LENGTH;
  }
}
```

### 패키지
- 패키지는 주로 개발 회사의 도메인 이름의 역순으로 만든다. 예) mycompany.com -> com.mycompany, apache.org -> org.apache
- 패키지는 클래스를 식별하는 용도로 사용되기 때문에 클래스의 전체 이름에 포함된다. 예) com.mycompany.Car
- 패키지 디렉토리는 클래스를 컴파일하는 과정에서 자동으로 생성 됨. 컴파일러는 클래스의 패키지 선언을 보고 디렉토리 자동 생성. 패키지 선언은 항상 소스 최상단에 위치해야 함.
- 패키지 이름은 모두 소문자로 작성하는 것이 관례이며 마지막에는 프로젝트 이름을 붙여주는 것이 일반적. 예) com.lg.projectname
- import한 두 패키지에 동일한 클래스 이름이 존재할 경우 해당 클래스를 사용 시 전체 이름을 명시해줘야 함.
  ```java
  import com.hankook.*;
  import com.kumho.*;
  public class Car {
    Tire tire = Tire(); //컴파일 에러 발생, com.hankook, com.kumho 패키지 안에 동일한 Tire 클래스가 존재하는 경우
    com.hankook.Tire tire = new com.hankook.Tire(); //풀네임 명시해줘야 함
  }
  ```

### 접근 제한
- 클래스 : public으로 선언하면 다른 패키지에서도 사용 가능. 클래스 선언 시 public 생략하면 default 접근 제한으로 같은 패키지에서만 접근 가능
- 생성자,필드,메소드 접근 제한
  |||
  |--|--|
  |public|모든 패키지에서 사용 가능|
  ||같은 패키지에서만 사용 가능|
  |private|클래스 내부에서만 사용 가능|
