# 스트림

## 스트림
- 컬렉션 및 배열의 요소를 반복 처리하기 위해 스트림 사용할 수 있음.
- 내부반복자이므로 Iterator보다 처리 속도가 빠르고 병렬 처리에 효율적    
  *외부반복자 : for문과 iterator과 같이 컬렉션의 요소를 컬렉션 바깥쪽으로 반복해서 가져와 처리    
   내부반복자 : 요소 처리 방법을 컬렉션 내부로 주입시켜서 요소 반복 처리 
- 람다식으로 다양한 요소 처리 정의할 수 있고 중간 처리, 최종 처리를 수행하도록 파이프 라인 형성할 수 있음.
```java
Set<String> set = new HashSet<>();
set.add("Hong");
set.add("Shin");
Stream<String> stream = set.stream(); //Stream 객체 얻음
stream.forEach(name->System.out.println(name));
```
### 내부 반복자
- 개발자 코드에서 제공한 데이터 처리 코드(람다식)을 가지고 컬렉션 내부에서 요소 반복 처리
- 멀티 코어 CPU를 최대한 활용하기 위해 요소들을 분배시켜 병렬 작업 가능
```java
List<String> list = new ArrayList();
list.add("Hong");
list.add("Kim");
Stream<String> parallelStream = list.parallelStream(); //병렬 처리 스트림 얻음
parallelStream.forEach(name-> {
    System.out.println(name+": "+Thread.currentThread().getName())
});
/* 출력. 병렬로 할 경우 요소의 순서는 다를 수 있다.
Hong : main
Kim : ForkJoinPool.commonPool-worker-1
*/
```

### 스트림 파이프라인
- 스트림은 하나 이상 연결될 수 있음
- 중간스트림에서 필터링, 매핑,정렬 등을 수행. 최종 처리는 요소 반복, 카운팅, 총합, 평균 등의 작업 수행
- 스트림 파이프라인 구성할 때는 반드시 최종 처리 부분이 있어야 함.
```java
//Student 스트림 얻기 -> score 스트림으로 변환 -> score 평균 구함
Stream<Student> studentStream = list.stream();
IntStream scoreStream = studentStream.mapToInt( student -> student.getScore());
double avg = scoreStream.average().getAsDouble();

//메소드 체이닝 패턴 이용
double avg = list.stream()
            .mapToInt(student->student.getScore())
            .average().getAsDouble();
```

- Stream(객체 요소 처리), IntStream, LongStream, DoubleStream..

### 컬렉션으로부터 스트림 얻기

## 요소 걸러내기(필터링)
필터링 메소드는 distinct(), filter()가 있음

### distinct()
- 요소의 중복을 제거
- 객체 스트림(Stream)의 경우 equals() 메소드의 리턴값이 true이면 동일 요소로 판단

### filter()
- 매개값으로 주어진 Predicate가 true를 리턴하는 요소만 필터핑
- Predicate는 함수형 인터페이스. Predicate<T>, IntPredicate, LongPredicate, DoublePredicate
- Predicate는 매개값을 조사한 후 boolean을 리턴하는 test() 메소드를 가진다
```java
//이름 중복 제거, 성이 Kim인 이름만 필터링
List<String> list = new ArrayList<>();
```

## 매핑
- 스트림의 요소를 다른 요소로 변환하는 중간 처리 기능.
- mapXXX(), asDoubleStream(), boxed(),flatMapXXX()...
- map(Function<T,R>)은 T를 R로 변환하여 Stream\<R\> 리턴한다.
  - 매개타입 Function은 함수형 인터페이스    
  - mapToInt(ToIntFunction\<T\>) : IntStream 리턴    
  - mapToObj(IntFunction\<U\>) : int를 U로 변환, Stram\<U\> 리턴
  - asLongStream() : int->long, LongStream 리턴
  - boxed() : Wrapper객체로 변환. ex) int->Integer, Stream<Integer> 리턴

### Comparable 구현 객체 정렬

```java
public class Student implements Comparable<Student>{
  private String name;
  private int score;

  public Student(String name, int socre){
    this.name = name;
    this.score = score;
  }

  public String getName() { return name;}
  public int getScore() { return score;}

  @Override
  public int compareTo(Student o){
    return Integer.compare(score, o.score); //compare(x,y) 같으면 0, x<y 음수, x>y 양수 
  }
}

public class Example {
  public static void main(String[] args){
    List<Student> studentList = new ArrayList<>();
    studentList.add(new Student("Hong", 30));
```

### 요소 조건 만족 여부(매칭)
```java
//모든 요소가 만족하는지 여부
boolean allMatch(Predicate<T> predicate);
//최소한 하나의 요소가 만족하는지 여부
boolean anyMatch(Predicate<T> predicate);
//모든 요소가 만족하지 않는지 여부
boolean noneMatch(Predicate<T> predicate);

//모든 요소가 2의 배수이면 true
int[] intArr = {2,4,6};
boolean result = Arrays.stream(intArr).allMatch(a->a%2==0);

//하나라도 3의 배수가 있으면 true
result = Arrays.stream(intArr).anyMatch(a->a%3==0);

//3의 배수가 하나도 없으면 true
result = Arrays.stream(intArr).noneMatch(a->a%3==0); 
```

### grouping
- collect()메소드는 Collection의 요소들을 그룹핑해서 Map객체를 생성하는 기능도 제공
- Collectors.groupingBy()메소드에서 얻은 Collector를 collect()메소드에 제공
- groupingBy는 Function을 이용해서 T를 K로 매핑하고, K를 키로 해 List<T>를 값으로 갖는 Map 컬렉션을 생성
``` 
Collector<T,?,Map<K,List<T>>> groupingBy(Function<T,K> classifier)
```
```java
//성별로 grouping하고 성별이 M인 학생 List 출력
List<Student> totalList = new ArrayList<>();
totalList.add(new Student("Hong","M", 23));
totalList.add(new Student("Lee","M", 24));
totalList.add(new Student("Kim","F", 21));
Map<String, List<Student>> map = totalList.stream().collect(
  Collectors.groupingBy(s->s.getSex())
);
List<Student> maleList = map.get("M");
maleList.stream().forEach(s->System.out.println(s.getName()));
```

### Parallel Operation
멀티스레드는 Concurrency 또는 Parallelism으로 실행 됨
- Concurrency : 하나의 코어에서 멀티 스레드가 번갈아 가며 작업 실행
- Parallelism : 멀티 코어를 각각 이용해서 병렬로 실행
  - Data parallelism : 전체 데이터 분할. 서브 데이터셋들을 병렬 처리
  - Task parallelism : 서로 다른 작업을 병렬 처리.

### 포크조인 프레임워크
자바 병렬 스트림은 포크조인 프레임워크 사용.    
- 포크 단계에서 요소 분할하고 각각의 서브 요소셋을 멀티코어에서 병렬 처리. 조인 단계에서 서브 결과 결합해서 결과 만듦.
- 병렬 처리 위해 스레드풀 사용. ExecutorService의 구현 객체인 ForkJoinPool 사용해서 작업 스레드 관리

### 병렬 스트림 사용
- parallelStream() 메소드는 Collection으로 부터 병렬 스트림 리턴
- parallel() 메소드는 기존 스트림을 병렬 처리 스트림으로 변환
|Return Type|Method|Collection or Stream|
|--|--|--|
|Stream|parellelStream()|List or Set Collection|
|Stream|parallel()|java.util.Stream|
```java
Random random = new Random();
List<Integer> scores = new ArrayList<>();

for(int i=0; i<100000000; i++){
  scores.add(random.nextInt(101)); //0이상 100이하 중 랜덤한 값
}
double avg = 0.0;
long startTime = 0;
long endTime = 0;
long time = 0;

Stream<Integer> stream = scores.stream();
startTime = System.nanoTime();
avg = stream.mapToInt(i -> i.intValue())
            .average()
            .getAsDouble();
endTime = System.nanoTime();
time = endTime - startTime;
System.out.println("avg : " + avg + ", 일반 스트림 처리 시간 : "+time+"ns");

Stream<Integer> parallelStream = scores.parallelStream();
startTime = System.nanoTime();
avg = parallelStream
          .mapToInt(i-> i.intValue())
          .average()
          .getAsDouble();
```

### 병렬 처리 성능
스트림 병렬 처리가 스트림 순차 처리보다 항상 실행 성능이 좋은 것은 아님. 병렬처리는 포크 및 조인, 스레드 풀 생성 추가 비용이 발생하기 때문임.
- 요소 수와 요소당 처리 시간 : Collection의 전체 요소 수가 적고 요소당 처리 시간이 짧으면 일반 스트림이 더 빠를 수 있음
- 스트림 소스의 종류
  - ArrayList와 배열은 인덱스로 요소를 관리하기 때문에 포크 단계에서 요소 분리가 쉬워 병렬 처리 빠름.
  - HashSet,TreeSet,LinkedList는 요소 분리가 쉽지 않으므로 병렬 처리가 상대적으로 늦음
- 코어의 수 : 수가 많을수록 병렬 스트림 성능 좋아짐. 코어 수가 적을 경우 병렬 스트림 스레드 수가 증가하여 동시성(Concurrency)이 많이 일어나므로 오히려 느려짐.
