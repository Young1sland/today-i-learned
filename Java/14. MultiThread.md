# Multi Thread

## Multi Thread
- 운영체제는 실행 중인 프로그램을 프로세스로 관리
- 하나의 프로세스가 멀티 태스킹이 가능한 이유는 멀티 스레드가 있기 때문.    
  스레드는 코드의 실행 흐름을 말하며 두개의 스레드가 있다면 두개의 실행 흐름 존재
- 멀티 프로세스는 서로 독립적이므로 하나의 프로세스에 오류가 발생해도 다른 프로세스에 영향 없다. 하지만 멀티 스레드는 프로세스 내부에서 생성되므로 하나의 스레드에서 예외 발생하면 프로세스가 종료 됨.

## 메인 스레드
- 메인 스레드는 main()메소드를 실행하면서 시작되며 추가 작업 스레드 만들어 실행 가능.
- 멀티 스레드에서는 실행 중인 스레드가 하나라도 있다면 프로세스 종료 되지 않음.

## 작업 스레드 생성과 실행
- 자바는 작업 스레드도 객체로 관리하므로 클래스 필요. Thread 또는 하위 클래스로 생성
```java
/**
  Thread 클래스로 직접 생성
  Thread thread = new Thread(Runnable target);
  Runnable은 스레드가 작업 실행할 때 사용하는 인터페이스
*/
class Task implements Runnable {
    @Override
    public void run(){
        //실행 코드
    }
}
Runnable task = new Task();
Thread thread = new Thread(task);
thread.start();

//익명 구현 객체 사용
Thread thread = new Thread(new Runnable(){
    @Override
    public void run(){
        //실행 코드
    }
});
thread.start();
```

```java
//Thread 자식 클래스로 생성
public class WorkerThread extends Thread {
    @Override
    public void run(){
        //실행 코드
    }
}
Thread thread = new WorkerThread();
thread.start();

//익명 자식 객체 사용
Thread thread = new Thread(){
    @Override
    public void run(){
        //실행 코드
    }
};
thread.start();
```

## 스레드 이름
메인 스레드는 main이라는 이름을 가지고 작업 스레드는 자동으로 'Thread-n'이란 이름 가진다. 스레드 이름 변경 가능하며 주로 디버깅에 사용된다.
```java
//작업 스레드 이름 설정
thread.setName("threadName");
//현재 코드를 어떤 스레드가 실행하고 있는지 확인하려면 정적 메소드 currentThread()로 스레드 객체 참조 얻은 다음 getName() 메소드로 이름 출력
Thread thread = Thread.currentThread();
String threadName = thread.getName();

public class ThreadExample{
    public static void main(String[] args){
        //객체 참조 얻기. 여기서는 main
        Thread mainThread = Thread.currentThread(); 
        System.out.println(mainThread.getName());

        Thread chatThread = new Thread(){
            @Override
            public void run(){
                //getName()은 Thread의 인스턴스 메소드
                System.out.println(getName());
            }
        };
        //작업 스레드 이름 변경
        chatThread.setName("chat-thread");
        chatThread.start();
    }
}
```

## 스레드 상태
- 스레드 객체를 생성(NEW)하고, start() 호출하면 실행 대기 상태(RUNNABLE)이 됨
- 실행 대기 스레드는 CPU 스케쥴링에 따라 CPU 점유하고 run() 메소드 실행 (RUNNING 상태)    
  스케쥴링에 의해 다시 실행 대기 상태로 돌아갈 수 있고 다른 스레드가 실행 상태가 됨.
- 실행 상태에서 run()메소드가 종료되면 스레드 실행 멈춤.(TERMINATED)
- 실행 상태에서 일시 정지 상태로 가기도 함. 다시 실행 상태가 되기 위해서는 실행 대기 상태로 가야 함.

|||
|:--:|:--:|
|sleep(long mills)| Thread의 static 메소드. 주어진 시간 동안 일시 정지로 만듬. 주어진 시간 지나면 자동으로 실행 대기 상태가 됨.|
|join()|호출한 스레드는 일시 정지 상태가 됨. 실행 대기 상태가 되려면 join()메소드를 가진 스레드가 종료되어야 함.|
|wait()|동기화 블록 내에서 스레드를 일시 정지 상태로 만듬|
|interrupt()|일시 정지 상태일 경우 InterruptedException 발생시켜 실행 대기 상태 또는 종료 상태로 만든다.|
|notify(), notifyAll()|wait()메소드로 인해 일시 정지 상태인 스레드를 실행 대기 상태로 만듬|
|yield()|실행 상태에서 다른 스레드에게 실행을 양보하고 실행 대기 상태가 됨|

### 주어진 시간 동안 일시 정지
Thread.sleep()을 이용하여 스레드를 일정 시간 동안 일시 정지 상태로 만들 수 있다.
```java
try { 
    Thread.sleep(3000);
}catch(InterruptedException e){
    //interrupt() 메소드 호출되면 실행
}
```

### 다른 스레드의 종료를 기다림
- 다른 스레드가 종료될 때까지 기다렸다가 실행해야 하는 경우 join() 사용.
- ThreadA가 ThreadB 종료 후 실행되야 하는 경우 ThreadA내에서 ThreadB.join() 호출    
  ThreadA는 ThreadB가 종료할 때까지 일시 정지 상태가 됨.
```java
//SumThread가 종료된 다음에 mainThread 실행
public class SumThread extends Thread {
    private long sum;
    public long getSum(){return sum;}
    @Override
    public void run(){
        for(int i=1;i<=100;i++){
            sum+=1;
        }
    }
}
public class JoinExample {
    public static void main(String[] args){
        SumThread sumThread = new SumThread();
        sumThread.start();
        try {
            sumThread.join(); //mainthread는 일시정지 상태가 됨
        } catch(InterruptedException e){}
        System.out.println("1-100 합: "+sumThread.getSum());
    }
}
```
### 다른 스레드에게 실행 양보
스레드는 반복적인 실행 위해 for,while문을 포함하는 경우가 많은데 무의미한 반복의 경우 다른 스레드에게 실행을 양보하고 자신은 실행 대기 상태로 가는 것이 성능에 도움이 될 수 있다.
이러한 경우 Thread.yield()를 사용한다.
```java
//처음 5초는 ThreadA,ThreadB 번갈아 가며 실행. 
//이후 ThreadA가 yield()호출하여 ThreadB가 더 많은 실행 기회 얻음
//10초 뒤에 다시 threadA,ThreadB 번갈아 가며 실행 됨
public class WorkThread extends Thread {
    public boolean work = true;
    public WorkThread(String name){
        setName(name); //thread 이름 설정하는 Thread 메소드
    }
    @Override
    public void run(){
        while(true){
            if(work){System.out.println(getName() + " is Working");}
            else {Thread.yield();}//일시 정지 상태로 가고 작업 양보
        }
    }
}
public class YieldExample {
    public static void main(String[] args){
        WorkThread workThreadA = new WorkThread("workThreadA");
        WorkThread workThreadB = new WorkThread("workThreadB");
        workThreadA.start();
        workThreadB.start();
        try {Thread.sleep(5000);} catch(InterruptedException e){}
        workThreadA.work = false;
        try {Thread.sleep(10000);} catch(InterruptedException e){}
        workThreadA.work = true;
    }
}
```

## 스레드 동기화

### 동기화 메소드 및 블록 선언
스레드가 사용 중인 객체를 다른 스레드가 변경할 수 없도록 synchronized로 잠금.    
synchronized는 인스턴스, 정적 메소드에 모두 가능. (객체를 잠근다..)
```java
//메소드 잠금
public synchronized void method(){
    //하나의 스레드만 사용 가능. 
}

//일부 블록에서만 잠금
public void method(){
    synchronized(객체){
        //하나의 스레드만 사용 가능
    }
}
```
```java
public class Calculator {
    private int memory;
    public int getMemory(){
        return money;
    }
    public synchorinzed void setMemory1(int memory){
        this.memory = memory;
        try {
            Thread.sleep(2000);
        }catch(InterruptedException e){
            System.out.println(Thread.currentThread().getName()+":"+this.memory);
        }
    }
    public void setMemory2(int memory){
        synchronized(this){
            this.memory = memory;
            try {
                Thread.sleep(2000);                
            }catch(InterruptedException e){}
            System.out.println(Thread.currentThread().getName()+":"+this.memory);
        }
    }
}

public class User1Thread extends Thread {
    private Calculator calculator;

    public UserThread(){
        setName("User1Thread");
    }
    public void setCalculator(Calculator calculator){
        this.calculator = calculator;
    }
    @Override
    public void run(){
        calculator.setMemory1(100);
    }
}

public class User2Thread extends Thread {
    private Calculator calculator;

    public UserThread(){
        setName("User2Thread");
    }
    public void setCalculator(Calculator calculator){
        this.calculator = calculator;
    }
    @Override
    public void run(){
        calculator.setMemory2(50);
    }
}

//user1Thread setMemory1() 실행하는 순간 Calculator 객체 잠금
//user2Thread는 객체가 잠금 해제될 때까지 Calculator 동기화 블록 실행 못함
//2초 뒤 잠금 해제되면 user2Thread는 setMemory2()의 동기화 블록 실행

public class SynchronizedExample {
    public static void main(String[] args){
        Calculator calculator = new Calculator();
        User1Thread user1Thread = new User1Thread();
        user1Thread.setCalculator(calculator)
        user1Thread.start();

        User2Thread user2Thread = new User2Thread();
        user2Thread.setCalculator(calculator)
        user2Thread.start();
    }

*/
```

### wait()과 notify()
두 개의 스레드가 교대로 번갈아 가며 실행이 필요한 경우    
- 공유 객체는 두 스레드가 작업할 내용을 각각 동기화 메소드로 정해 놓는다. 
- 한 스레드가 작업을 완료하면 notify() 메소드를 호출해서 일시 정지 상태에 있는 다른 스레드를 실행 대기 상태로 만듬. 자신은 wait() 메소드를 호출하여 일시 정지 상태로 만듬.
- wait(), notify()는 모두 동기화 메소드,블록 내에서만 사용 가능
- notify()는 wait()에 의해 일시정지된 스레드 중 한개만 대기 상태로 만듬. notifyAll()은 일시 정지된 모든 스레드를 실행대기 상태로 만듬
```java
public class WorkObject{
    public synchronized void methodA(){
        Thread thread = Thread.currentThread();
        System.out.println(thread.getName() + ": methodA");
        notify(); //다른 스레드를 실행 대기 상태로 만듬
        try{
            wait();//자신의 스레드는 일시 정지 상태로 만듬
        }catch(InterruptedException e){}
    }
     public synchronized void methodB(){
        Thread thread = Thread.currentThread();
        System.out.println(thread.getName() + ": methodB");
        notify(); //다른 스레드를 실행 대기 상태로 만듬
        try{
            wait();//자신의 스레드는 일시 정지 상태로 만듬
        }catch(InterruptedException e){}
    }
}

public class ThreadA extends Thread {
    private WorkObject workObject;
    public ThreadA(WorkObject workObject){
        setName("ThreadA");
        this.workObject = workObject;
    }
    @Override
    public void run(){
        for(int i=0;i<10;i++){
            workObject.methodA();
        }
    }
}
public class ThreadB extends Thread {
    private WorkObject workObject;
    public ThreadB(WorkObject workObject){
        setName("ThreadB");
        this.workObject = workObject;
    }
    @Override
    public void run(){
        for(int i=0;i<10;i++){
            workObject.methodB(); //둘다 methodA 불러도 같은데.. 굳이?
        }
    }
}

public class WaitNotififyExample {
    public static void main(String[] args){
        WorkObject workObject = new WorkObject();
        ThreadA threadA = new ThreadA(workObject);
        ThreadB threadB = new ThreadB(workObject);
        threadA.start();
        threadB.start();
    }
}
//출력 번갈아 가며 출력 됨
/*
..
ThreadA : methodA
ThreadB : methodB
..
*/
```
    
## 스레드 안전 종료
- stop()메소드가 있으나 갑자기 종료 시 리소스들이 불안전한 상태로 남겨지기 때문에 deprecated 되었음
- 스레드를 안전하게 종료하는 방법은 조건 이용 방법과 interrupt() 메소드 이용 방법이 있음

### 조건 이용
```java
public class ThreadA extends Thread {
    private boolean stop;
    public void setStop(boolean stop){
        this.stop = stop;
    }
    @Override void run(){
        while(!stop){
            //스레드 반복 실행 부분
        }
        //리소스 정리
        //실행 종료
    }
}

public class Example {
    public static void main(String[] args){
        ThreadA threadA = new ThreadA();
        threadA.start();

        try {
            Thread.sleep(3000);
        }catch(InterruptedException e){}
        threadA.setStop(true);        //종료 위해 stop 필드 변경 
    }
}
```
### Interrupt() 메소드 이용
- interrupt()메소드는 스레드가 일시정지 상태에 있을 때 InterruptedException 예외를 발생시킴
- 스레드가 실행 대기/실행 상태일 때는 interrupt() 메소드가 호출되어도 Exception 발생 안함.
- 일시 정지를 만들지 않고 interrupt() 메소드 호출 여부를 알 수 있음.
  ```java
  boolean status = Thread.interrupted(); //정적 메소드
  boolean status = thread.isInterrupted(); //인스턴스 메소드
  ```
```java
//Thread.sleep()으로 일시정지 만들어 InterruptedException
public class ThreadA extends Thread {
    @Override void run(){
        try{
            while(true){
                //스레드 반복 실행 부분
                Thread.sleep(1); //InterruptedException 발생할 수 있도록 일시 정지 만듦
            }
        } catch(InterruptedException e){}
        //리소스 정리
        //실행 종료
    }
}

public class Example {
    public static void main(String[] args){
        ThreadA threadA = new ThreadA();
        threadA.start();
        try {
            Thread.sleep(3000);
        }catch(InterruptedException e){}
        threadA.interrupt();        //interrupt 호출
    }
}
```
```java
//interrupted() 메소드 사용
public class ThreadA extends Thread {
    @Override void run(){
        while(true){
            //스레드 반복 실행 부분
            if(Thread.interrupted()){
                break;
            }
        }        
        //리소스 정리
        //실행 종료
    }
}
public class Example {
    public static void main(String[] args){
        ThreadA threadA = new ThreadA();
        threadA.start();
        try {
            Thread.sleep(3000);
        }catch(InterruptedException e){}
        threadA.interrupt();        //interrupt 호출
    }
}
```

## 데몬 스레드
