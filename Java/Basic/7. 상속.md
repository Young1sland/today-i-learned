### 생성자 호출
- 자식 생성자 호출 시 명시적으로 super를 호출하지 않으면 super()가 컴파일 과정에서 자동 추가 됨. 부모 클래스의 기본 생성자가 호출 됨.
- 부모 클래스에 기본 생성자가 없고 매개변수를 갖는 생성자만 있다면 개발자는 super(매개값, ...) 코드를 명시적으로 넣어줘야 함.
```java
//자식클래스 생성자
public class Parent {
    public Parent(String name){}
}
public class 
public 자식클래스() {
    //명시적으로 super()호출 하지 않으면 super()가 추가되며 부모클래스의 기본 생성자 호출 됨    
}
public 자식클래스(){
    super(매개값,..);//부모 클래스에 기본 생성자가 없고 매개변수를 갖는 생성자만 있다면 명시적으로 호출해줘야 함.
}
```

### 메소드 재정의(Override)
- @Override 생략 가능하지만 붙이면 컴파일 단계에서 정확히 Overriding되었는지 체크 가능.
- 자식 메소드에서 부모 메소드를 호출할려면 super.{methodName}형태로 호출 가능
```java
public class Airplane {
    public void fly(){}
}

public class SuperAirplane extends Airplane {
    public static final int flyMode = 1;
    @Override
    public void fly(){
        if(flyMode == 1){
            //
        } else {
            super.fly(); //부모 메소드 호출
        }        
    }
}
```

### final 클래스와 final 메소드
- final class는 부모 클래스가 될 수 없어 자식 클래스 만들 수 없음
- final method는 overriding 할 수 없음.

### 타입 변환
- 자동 타입 변환    
  ```부모타입 변수 = 자식타입객체;```
- 강제 타입 변환
  - 부모 타입은 자식 타입으로 자동 변환되지 않고 캐스팅 연산자로 강제 타입 변환할 수 있다.    
  ```자식타입 변수 = (자식타입) 부모타입객체 ```
  - 자식 객체가 부모 타입으로 자동 변환된 후 다시 자식 타입으로 변환할 때 강제 타입 변환을 사용할 수 있다.
  ```java
  class Parent {
    public void method1(){}
    public void method2(){}
  }
  class Child extends Parent {
    @Override
    public void method2(){}
    public void method3(){}
  }
  Child child = new Child();
  Parent parent = child; //parent == child 두 변수는 같은 객체를 참조한다.
  parent.method1(); //OK
  parent.method2(); //Override된 Child의 함수가 호출 됨
  parent.method3(); //Error

  Child child2 = (Child)parent; //강제 타입 변환
  child2.method3(); //OK
  ```

### 객체 타입 확인
변수가 참조하는 객체의 타입을 확인할 때 Instanceof연산자 사용(`객체 instanceof 타입`)
```java
//매개변수 입력 값이 Child이면 변환
public void method(Parent parent){
    if(parent instanceof Child){ //Child 타입인 경우만 Child로 강제 타입 변환하여 사용
        Child child = (Child) parent; 
    }

    //Java12부터 Instanceof 연산 결과가 true일 경우 타입 변환 없이 우측 타입 변수 사용 가능.
    if(parent instanceof Child child){
        //child 변수 사용 가능
    }
}
```

### 추상 클래스
- new 연산자를 사용할 수 없으며 새로운 실체 클래스를 만들기 위한 부모 클래스로만 사용된다.    
- 자식 객체가 생성될 때 super()로 추상클래스의 생성자가 호출되기 땜분에 생성자가 반드시 있어야 한다.
- 추상 클래스는 추상메소드를 선언할 수 있으며 abstract 키워드가 붙고 메소드 실행부{}가 없다.     
    자식 클래스에서 반드시 오버라이딩해야 한다.
```java
public abstract class Phone {
    String owner;

    public void sound(); //추상 메소드

    Phone(String owner){ //생성자 반드시 정의 필요
        this.owner = owner;
    }
    
    void TurnOn(){
        System.out.println("Turn On");
    }
}

public class SmartPhone extends Phone {
    SmartPhone(String owner){
        super(owner);
    }
    void search(){
        System.out.println("Search");
    }

    //추상메소드 오버라이드
    @Override
    public void sound(){
        System.out.println("Ping");
    }
}

public class Main {
    public static void main(String[] args){
        //Phone phone = new Phone(); new 연산자 사용X
        SmartPhone smartPhone = new SmartPhone("Kim");
        smartPhone.turnOn();
        smartPhone.search();

        phoneSound(new SmartPhone());
    }

    public static void phoneSound(Phone phone){
        phone.sound();
    }
}
```

### 봉인된 클래스(Sealed Class)
- 기본적으로 final 클래스를 제외한 모든 클래스는 부모 클래스가 될 수 있다.
- Java15부터는 무분별한 자식 클래스 생성을 방지하기 위해 Sealed 클래스가 도입되었다.
- sealed 키워드를 사용하면 permits 키워드 뒤에 상속 가능한 자식 클래스를 지정해야 한다.    
  sealed class를 상속하는 클래스는 final,non-sealed,sealed 키워드를 사용하여 선언해줘야 한다.
```java
//Employee, Manager만 상속 가능
public sealed class Person permits Employee, Manager {
    public String name;
    public void work(){
        System.out.println("person work");
    }
}

//final클래스로 선언. 이후 해당 클래스 extends 불가 
public final class Employee extends Person {
    @Override
    public void work(){
        System.out.println("Employee work");
    }
}

//non-sealed 클래스 선언. 해당 클래스를 모든 클래스가 extends 가능
public non-sealed class Manager extends Person {
    @Override
    public void work(){
        System.out.println("Manager work");
    }
}

//sealed 클래스. extends 제한
public sealed class SealedPerson extends Person {
    @Override
    public void work(){
        System.out.println("SealedPerson work");
    }
}

```
