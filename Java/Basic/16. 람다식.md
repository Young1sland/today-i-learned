# 람다식

## 람다식
### 함수형 프로그래밍
 - 순수함수(pure function)를 구현하고 호출함으로써 외부 자료에 Side Efeect를 주지 않도록 구현하는 방식. 순수함수란 매개변수만을 사용하여 만드는 함수.
 - 함수를 기반으로 하는 프로그래밍이고 입력받는 자료만 사용하여 병렬 처리 가능

### 람다식 표현
- (매개변수,..)->{처리 내용}
- 자바는 람다식을 익명 구현 객체로 변환한다.
```java
(int x, int y)-> {return x+y;}
str -> {System.out.println(str);} //매개 변수가 하나인 경우 자료형과 괄호 생략 가능
str -> System.out.println(str); //실행문이 한 문장인 경우 중괄호 생략 가능
//실행문이 한 문장의 반환문인 경우엔 return, 중괄호 모두 생략 가능
(x,y) -> x+y;
str -> str.length;
//매개 변수가 두개 이상인 경우 괄호를 생략할 수 없음
x,y -> {System.out.println(x+y);} //오류
//실행문이 한 문장이라도 return문은 중괄호를 생략 할 수 없음
str -> return str.length(); //오류

public interface Calcuable {
    void caculate(int x, int y);
}
public void action(Calculable calculable){
    int x = 10;
    int y = 4;
    calculable.calculate(x,y);
}
action( (x,y)-> {
    System.out.println(x+y);
});
```

### 함수형 인터페이스 선언
- 람다식 선언하기 위한 인터페이스
- 익명 함수와 매개 변수만으로 구현되므로 인터페이스는 단 하나의 메서드를 선언해야 함.
- @FunctionalInterface : 함수형 인터페이스임을 보장. 추상 메소드가 두개이상이면 에러 발생
```java
@FunctionalInterface
public interface Calculable {
    void calculate(int x, int y);
}
```

## 람다식 예제
```java
public class Button {
    @FunctionalInterfface
    public static interface ClickListener {
        void onClick();
    }
    private ClickListener clickListener;
    public void setClickListener(ClickListener clickListener){
        this.clickListener = clickListener;
    }
    public void click(){
        this.clickListener.onClick();
    }
}
public class ButtonExample{
    public static void main(String[] args){
        Button btnOk = new Button();
        btnOk.setClickListener(()->{
            System.out.println("Click");
        });
    }
}
```

## 메소드 참조
메소드를 참조하여 매개변수의 정보 및 리턴 타입을 알아내 람다식에서 불필요한 매개변수를 제거
```java
(left,right)-> Math.max(left,right);
//위의 람다식은 Math.max()의 매개값만 전달 함. 이 경우 메소드 참조를 이용해 표현 가능
Math :: max; 
```
### 정적 메소드와 인스턴스 메소드 참조
- 정적 메소드 참조할 경우 클래스 이름 뒤에 '::'와 메소드 이름 붙임 ( 클래스 :: 메소드 )
- 인스턴스 메소드의 경우 객체를 생성한 다음 참조 변수 뒤에 '::'와 메소드 이름 붙임( 참조변수 :: 메소드)
```java
@FunctionalInterface
public interface Calcuable {
    double calc(double x, double y);
}
public class Person {
    public void action(Calculable calcuable){
        double result = calculable.calc(10,4);
        System.out.println(result);
    }
}
public class Computer {
    public static double staticMethod(double x, double y){
        return x+y;
    }
    public double instanceMethod(double x, double y){
        return x*y;
    }
}
public class Example {
    public static void main(String[] args){
        Person person = new Person();
        //정적 메소드일 경우
        //person.action((x,y)-> Computer.staticMethod(x,y));
        person.action(Computer :: staticMethod);

        //인스턴스 메소드일 경우
        Computer com = new Computer();
        //람다식
        //person.action((x,y)-> com.instanceMethod(x,y));
        person.action(com :: instanceMethod);
    }
}
```

### 매개변수의 메소드 참조
- 람다식에서 제공되는 a 매개변수의 메소드를 호출해서 b 매개변수를 매개값으로 사용
- a의 클래스 이름 뒤에 '::'와 메소드 이름 붙임
```(a,b) -> {a.instanceMethod(b);} => a의 클래스 이름 :: instanceMethod```
```java
@FunctionalInterface
public interface Comparable {
    int compare(String a, String b);
}
public class Person {
    public void ordering(Comparable comparable){
        String a = "Hong";
        Sting b = "Kim";
        int result = comparable.compare(a,b);
        System.out.println(result);
    }
}
public class Example {
    public static void main(String[] args){
        Person person = new Person();
        //person.ordering((a,b)-> a.compareToIgnoreCase(b))
        person.ordering(String :: compareToIgnoreCase);
    }
}
```

### 생성자 참조
- 람다식이 단순히 객체를 생성하고 리턴한다면 생성자 참조로 대치 가능
- 생성자가 여러개가 있을 경우 컴파일러는 동일한 매개변수 타입,개수 가지는 생성자 찾아 실행. 존재하지 않으면 에러 발생
```(a,b)-> {return new 클래스(a,b);} => 클래스 :: new```
```java
@FunctionalInterface
public interface Creatable1 {
    public Member create(String id);
}
public interface Creatable2
{
    public Member create(String id, String name);
}

public class Member {
    private String id;
    private String name;
    public Member(String id){
        this.id = id;        
    }
    public Member(String id, String name){
        this.id = id;
        this.name = name;
    }
}

public class Person {
    public Member getMember1(Creatable1 creatable){
        String id = "winter";
        Member member = creatable.create(id);
        return member;
    }
    public Member getMember2(Creatable2 creatable){
        String id = "winter";
        String name = "Kim";
        Member member = creatable.create(id,name);
        return member;
    }
}
public Example {
    public static main(String[] args){
        Person person = new Person();
        Member m1 = person.getMember1(Member :: new); //Member(id)
        Member m2 = person.getMember2(Member :: new); //Member(id,name)
    }
}
```
