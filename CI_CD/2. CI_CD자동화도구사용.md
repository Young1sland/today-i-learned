### 플러그인 설치 확인
1. Jenkins 관리 -> Plugin Manager
   - github Integration,maven Integration 확인
2. Jenkins 관리 -> Global Configuration    
   - docker 접속하여 git, mvn 명령어 사용가능한지 확인
   - 안될 경우 설치 필요. Global Configuration에서 git,Mvn 탭에서 설치 설정

### 실습 Git에서 코드 가져와서 빌드 하기
1. 새로운 아이템 -> Maven Project (없다면 Maven 설치 확인할 것)
2. 소스 코드 관리 탭 -> git 선택 -> 대상 repository URL 입력    
   https://github.com/joneconsulting/cicd-web-project
3. 소스 코드 관리 탭 -> Branches to build -> branch 이름이 일치하는지 확인
4. Build 탭 -> Root POM pom.xml -> Goals and options : clean compiler:compile package    
   - 빌드 이력 있으면 지움(clean). 빌드(compile). 패키지 파일 생성(패키지 파일 형식은 pom.xml의 packaging으로 결정)
5. 지금 빌드 -> 로그 확인
6. 작업 공간 탭으로 가서 파일 확인, docker 접속하여 target 폴더의 war 확인
   ```shell
   $ docker exec -it jenkins-server bash # /var/jenkins_home/workspace/
   ```

### Tomcat 웹서버에 배포하기
Tomcat 웹서버에 deploy해본다.
1. deploy 플러그인 설치(war 파일을 톰캣에 복사하기 위한 플러그인)  
   Manage Jenkins → Jenkins Plugins → available → deploy to container plugin 설치
  
2. 새로운 아이템 → Maven Project
3. 소스 코드 관리 탭 -> git 선택 -> 대상 repository URL 입력    
   https://github.com/joneconsulting/cicd-web-project
4. Build 탭 -> Root POM pom.xml -> Goals and options : clean compile package
5. Post-build Actions(빌드 후 조치) → Deploy war/ear to a container → WAR/EAR files : **/*.war(하위 폴더 포함 war 확장자 가진 파일이 대상)    
   → Containers : Tomcat 9.x Remote → Add credentials
6. Add Tomcat Credentials → username : deployer → ID : deployer_user → Save   
Tomcat에서 설정한 사용자 계정으로 추가 후 사용한다. 
   ```xml
   <!-- conf/tomcat-users.xml -->
   <user username="admin" password="admin" roles="manager-gui, manager-script, manager-jmx, manager-status"/>
   <user username="deployer" password="deployer" roles="manager-script"/>
   <user username="tomcat" password="tomcat" roles="manager-gui"/>
   ```
7. 생성한 Credentials 선택 : deployer → Tomcat URL : http://{자신의 IP 주소}/8081
   > 이 때 localhost로 URL을 설정하면 도커로 실행 중인 젠킨스는 컨테이너 내부에서 포트를 찾으려고 하므로 에러가 발생한다. 그러므로 자신의 IP주소로 설정해준다.
8. 지금 빌드 → 로그 확인
9. tomcat/webapps/hello-world.war,폴더 생성 확인
10. http://localhost:8081/manager/html에 app 확인
11. http://localhost:8081/hello-world/ 접근 확인

### PollSCM 설정을 통한 지속적인 파일 업데이트
1. Project > Configure > Build Triggers
   - Build Periodically : cron으로 동작. 변경사항이 없어도 빌드
   - Poll SCM : cron으로 동작. commit이 있을 때만 빌드
2.  Build Triggers →  Poll SCM, Schedule * * * * * (매초 동작)
3.  git에 commit을 push
4.  jenkins 로그에서 빌드가 자동으로 발생됨을 확인

### 외부 서버에 Docker Container로 배포하기
1. ssh 플러그인 설치 - Publish Over SSH(ssh 통한 파일 복사 위한 플러그인)
2. ssh server 추가
   - Jenkins 관리 → System → Publish over SSH 탭 → Add SSH Servers
   - hostname 입력. 만약 local의 서버를 사용 중이라면 localhost가 아닌 본인의 IP로 설정
   - 고급 > Use password authentication 설정. username, password 입력
   - port 입력
   - Test Configuration -> 성공 확인 후 저장
3. 새로운 Item 생성
   - Copy from으로 기존 Item의 설정을 가져올 수 있다. 새로운 Item > Copy from > item 이름 선택
4. 구성 설정
   - Poll SCM 체크되어 있다면 해제
   - 빌드 후 조치(Post-build Actions)
     - Deploy war/ear to a container 설정되어 있다면 제거
     - Send build artifacts over SSH 설정
       - Name : Publish over ssh에서 설정한 이름
       - Transfer set
       - Source files : target/*.war (서버에 복사될 파일들)
       - Remove prefix : target (삭제하고자하는 prefix)
       - Remote directory: . (현재 디렉토리)
       - Exec Commnad : (이후 실행한 command)

5. 지금 빌드
6. SSH server의 폴더에 war 파일이 정상적으로 복사되었는지 확인
7. SSH 서버에서 도커 실행 위한 Dockerfile을 생성
   ```Dockerfile
   FROM tomcat:9.0

   LABEL org.opencontainers.image.authors="edowon0623@gmail.com"
   # hello-world.war를 복사
   COPY ./hello-world.war /usr/local/tomcat/webapps
   ```
8. SSH 서버에서 docker image build/run 되는지 사전 확인
   ```shell
   $ docker build -t docker-server -f Dockerfile . # docker image 생성
   $ docker run -p 8080:8080 --name mytomcat docker-server:latest # 실행

   # 확인 후 container/image 삭제
   $ docker stop mytomcat # 중단
   $ docker rm mytomcat # container 삭제
   $ docker rmi docker-server # 생성한 docker image 삭제
   ```
9. Exec Command에 docker image build/run하는 명령 추가
   ```
   docker build --tag=cicd-project -f Dockerfile .;
   docker run -d -p 8080:8080 --name mytomcat cicd-project:latest

   ```
10. 지금 빌드 후 image 생성/실행 확인. 
11. 웹 접근 확인 http://localhost:8082/hello-world/

다시 빌드하면 오류 발생. 컨테이너가 실행 중인 상태에서 같은 이름의 컨테이너으로 다시 같은 동작을 시도하기 때문임.
