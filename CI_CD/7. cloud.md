### AWS AMI 생성
java 설치 후 AMI 생성해둔다. 5개 EC2 생성(ec2, tomcat, ansible, sonarqube, jenkins)
```sh
#  aws coreetto 다운로드

# jdk11 설치
$ sudo yum localinstall aws_corretto_jdk11.rpm

#java 버전 확인
$ java -version
$ javac -version

# 다운받은 패키지 삭제
$ rm -rf aws_corretto_jdk11.rpm

# 설치한 자바 버젼을 선택하여 설정
$ sudo /usr/sbin/alternatives --config java
```

#### user,role 설정
`$ vi conf/tomcat-users.xml`
```xml

<role rolename="manger-script"/>
<role rolename="manager-jmx"/>
<role rolename="manager-status"/>
<user username="admin" password="admin1234" roles="manager-gui, manager-script, manager-jmx, man
ager-status"/>
<user username="tomcat" password="tomcat" roles="manager-gui"/>

```
#### 설정 적용 위한 재시작
```sh
$ ./bin/shutdown.sh
$ ./bin/startup.sh
```
http://43.200.191.249:8080/manager/html 다시 접근해보자.

#### Tomcat Role
- manager-gui: 관리자 앱 웹 기반 GUI 접근
- manager-script: 관리자 앱 스크립트 인터페이스 접근
- manager-jmx: JMX Proxy 서블릿에 접근   
  JMX (Java Management Extensions): 애플리케이션, 시스템, 네트워크 등을 관리 및 모니터링하기 위함
- manager-status: 서버 상태 페이지 접근

### Ansible 서버 설치하기
```sh
$ sudo yum install -y ansible
```
#### hosts 파일 설정. 없다면 생성
그룹 이름,private ip로 설정
```
[localhost]
localhost

[docker]
172.31.28.108

[tomcat]
172.31.16.166
```
#### Ansible은 다른 hosts 접근 시 SSH 사용함. 접근 허용하기 위해 ansible에서 key생성하고 이를 docker,tomcat 서버 authorized_keys에 추가해주자.
```sh
# Ansible 서버
$ ssh-keygen -t rsa
```
Ansible에서 생성한 publicKey를 .ssh/id_rsa.pub를 복사하여 docker,tomcat 서버의 .ssh/authorized_keys에 추가. 
localhost에 접근 시에도 key가 등록되어 있어야 한다. 똑같이 authorized_keys에 등록해주자.  
Ansible 접근 확인해 보자.
```sh
# 접근 확인
$ ssh ec2-user@172.31.28.108
$ ssh ec2-user@172.31.16.166

# ansbile로 접근 확인
# ansible all -m ping
$ ansible docker -m ping
$ ansible tomcat -m ping
$ ansible localhost -m ping

    "ansible_facts": {
        "discovered_interpreter_python": "/usr/bin/python3.9"
    },

    }
```

### SonarQube 설치하기
SonarQube는 t2.micro로는 동작하지 않으므로 tier를 높이자.
```sh
$ cd /opt
$ mkdir sonarqube
# 필요한 기능은 v7.x대로 충분
$ sudo wget https://binaries.sonarsource.com/Distribution/sonarqube/sonarqube-7.6.zip
$ sudo unzip sonarqube-7.6.zip
```
#### ec2-user 권한에 소나큐브 사용할 수 있는 모든 권한 소유권 이전
- sonarQube는 내부적으로 Elasticsearch를 사용하여 분석 결과 데이터를 저장하고 관리하고 있음.
- 루트 권한 아닐 경우 오류가 발생할 수 있으므로 소유권 이전 필요.
- sonarqube 폴더 전체의 소유자 변경. root -> ec2-user
```sh
$ sudo chown -R ec2-user:ec2-user /opt/sonarqube/
```

#### 실행
```sh
# port 등의 설정은 하기 파일에서 설정. 소나큐브 디폴트 포트는 9000.  냅두자.
$ vi conf/sonar.properties

# 실행
$ ./bin/linux-x86-64/sonar.sh start
# 상태체크
$ ./bin/linux-x86-64/sonar.sh status
    SonarQube is running (3339).
```
