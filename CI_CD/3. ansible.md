### Infrastructure as Code(IaC)
- 시스템,하드웨어 또는 인터페이스의 구성 정보를 파일(스크립트)을 통해 관리 및 프로비저닝
- IT 인프라스트럭처, 베어 메탈 서버 등의 물리 장비 및 가상 머신과 관련된 구성 리소스를 관리
- 버전 관리를 통한 리소스 관리
    > 베어 매탈 : 어떠한 운영 체제나 소프트웨어가 설치되지 않은 순수한 하드웨어 상태를 의미   
    > 베어 메탈 서버 : 베어 메탈 서버는 가상화 레이어나 하이퍼바이저 없이 직접 물리적 서버 하드웨어에 운영 체제가 설치되는 서버를 의미

### Ansible
- https://docs.ansible.com/ansible
- 여러개의 서버를 효율적으로 관리할 수 있게 해주는 환경 구성 자동화 도구
- client node들은 별도의 Agent 설치 없이 python 모듈,ssh 가지면 연결 가능함.
- 환경 설정 파일 : /etc/ansible/ansible.cfg
- Ansible에서 접속하는 호스트 목록 : /etc/ansible/hosts

### docker로 ansible 실행하기
```sh
$ docker pull edowon0623/ansible-server:m1
$ docker run --privileged --name ansible-server -itd -p 20022:22 -p 8081:8080 \
 -e container=docker -v /sys/fs/cgroup:/sys/fs/cgroup --cgroupns=host edowon0623/ansible-server:m1 /usr/sbin/init

# docker로 실행한 모든 컨테이너는 bridge network driver에 연결되어 있음. Containers에서 정보 체크

    "Containers": {
        "Name": "ansible-server"
        "IPV4Address":172.XX.XX.X,
        ..
        "Name": "jenkins-server",
        "IPV4Address":172.XX.XX.X,
    },

$ ssh root@localhost -p 20022 # or docker exec -it ansible-server bash, pw : P@ssw0rd

# ssh 접속 후
$ hostname -i # 호스트 이름을 ip로 확인, -i 옵션 없으면 원래 hostname 출력

$ systemctl status docker

# 참고 : clear 명령어 안될 때

$ vi /etc/ansible/hosts

# Ansible에서 다른 서버에 password 입력 없이 접근 위해 키를 만들고 public key를 상대 서버에 배포해 본다.
$ ssh-keygen
$ ssh-copy-id root@172.XX.XX.X # SSH 공개 키를 원격 서버의 ~/.ssh/authorized_keys 파일에 복사. id_rsa.pub를 복사
$ ssh root@172.XX.XX.X # password 입력 없이 접근 가능
```

### Ansible 기본 명령어
Ansible 설정은 멱등성을 가지며 동일한 설정이 여러번 있어도 한번만 적용된다.
> 멱등성(Idempotence) : 특정 연산을 한 번 이상 적용해도 결과가 변하지 않는 성질
#### 실행 옵션
```
-i(--inventory-file) : 적용될 호스트들에 대한 파일 정보
-m(--module-name) : 모듈 선택
-k(--ask-pass) : 관리자 암호 요청
-K(--ask-become-pass) : 관리자 권한 상승
--list-hosts : 적용되는 호스트 목록
```
```sh
$ ansible all -m ping # all은 /etc/ansible/hosts에 설정된 모든 그룹에 대해 실행. all 대신 그룹명 지정 가능, -m 모듈 선택
$ ansible all -m shell -a "free -h" # -a(arguments) 모듈에 인자 전달. 'free -h' shell command를 실행. 
# free : 시스템 메모리 정보. -m MB로 표시, -h 사람이 읽기 쉬운 형식으로 표시(1K,20M 등)

$ ansible all -m copy -a "src=./test.sh dest=/tmp" # copy하기
$ ansible devops -m yum -a "name=httpd state=present" # devops 그룹에 httpd 패키지 설치.
# 대상 host에 접근하여 설치 확인
$ systemctl status httpd
```

### Ansible Playbook
사용자가 원하는 내용을 미리 작성해 놓은 파일로 다수의 서버에 반복 작업을 처리하는 경우 사용
#### 파일에 특정 내용 추가하는 예제   
1. yml 작성
    ```yml
    --- # 구분자
    - name: Add an ansible hosts # 이름 지정
        hosts: localhost # 대상 호스트 지정, 자신을 대상으로 지정
        tasks: # 작업 정의
        - name: Add an ansible hosts # 수행 작업 이름
            blockinfile: # blockinfile 모듈 사용. 파일에 특정 블록의 내용을 추가하거나 수정
            path: /etc/ansible/hosts
            block: | # 추가하려는 내용의 블록을 지정, 파이프(|) 기호는 여러 줄의 내용을 나열할 수 있게 해줌
                [mygroup]
                172.17.0.5
    ```
2. ansible-playbook 명령어로 yml 실행
```sh
$ ansible-playbook first-playbook.yml
$ cat /etc/ansible/hosts # playbook으로 추가된 블록의 경우 # BEGIN ANSIBLE MANAGED BLOCK 주석이 추가 됨.
    [devops]
    172.17.0.4
    172.17.0.3    
    # BEGIN ANSIBLE MANAGED BLOCK
    [mygroup]
    172.17.0.5
    # END ANSIBLE MANAGED BLOCK
```
#### 파일 복사 예제
1. yml 작성
    ```yml
    - name: Ansible Copy Example Local to Remote
      hosts: devops
      tasks:
        - name: copying file with playbook
          copy: # copy 모듈 사용
            src: ~/sample.txt
            dest: /tmp
            owner: root
            mode: 0644
    ```
2. ansible-playbook 실행   
    
   ```sh
   $ ansible-playbook playbook-sample1.yml
   ```

#### 파일 다운로드하는 예제
1. yml 작성
    ```yml
    - name: Download Tomcat9 from tomcat.apache.org
      hosts: devops
      tasks:
        - name: Create a Directory /opt/tomcat9
          file: # file module 사용, 디렉토리 생성
            path: /opt/tomcat9
            state: directory
            mode: 0755
        - name: Download Tomcat using get_url
          get_url: # get_url 사용
            url: https://dlcdn.apache.org/tomcat/tomcat-9/v9.0.82/bin/apache-tomcat-9.0.82.tar.gz
            dest: /opt/tomcat9
            mode: 0755
            checksum: sha512:https://downloads.apache.org/tomcat/tomcat-9/v9.0.82/bin/apache-tomcat-9.0.82.tar.gz.sha512
    ```
2. ansible-playbook 실행   
   ```sh
   $ ansible-playbook playbook-sample2.yml
   ```
