### 데이터베이스의 종류
- 계층형 : 트리 형태의 자료구조에 데이터 저장. 1대N 관계 표현. 부모와 자식 관계 표현 쉬움
- 네트워크형 : 오너와 멤버 형태로 데이터 저장. 1대N, M대N 표현 가능
- 관계형 : 릴레이션에 데이터 저장하고 관리. 집합,관계 연산 가능

### 관계형 데이터베이스의 집합 연산과 관계 연산
- 집합 연산 : 합집합(Union),차집합(Difference), 교집합(Intersection), 곱집합(Cartesian product)
- 관계 연산
  - 선택(Selection)
  - 투영(Projection): 조건에 맞는 속성만 조회
  - 결합(Join)
  - 나누기(Division) : 기존 릴레이션에서 나누는 릴레이션이 가지고 있는 속성과 동일한 값을 가지는 행(튜플)을 추출하고 나누는 릴레이션의 속성을 삭제한 후 중복 행 제거하는 연산

### 테이블 구조
- primary key : 유일성,최소성, Not Null 만족하면 해당 테이블 대표하는 것
- 테이블은 행과 칼럼으로 구성 됨. 행은 튜플이라고도 함. 컬럼은 필드,속성이라고도 함.

### SQL(Structed Query Language)
SQL은 ANSI.ISO 표준 준수하기 때문에 DBMS가 변경되어도 그대로 사용할 수 있다.
- 종류 : DDL(테이블 생성,변경,삭제), DML(입력,수정,삭제,조회), DCL(사용자 권한 부여,회수), TCL(트랜잭션 제어)
- 트랜잭션 : 데이터베이스의 작업을 처리하는 단위
  - 특성
    - 원자성(Atomicity) :  연산이 전부 실행되거나 전혀 실행되지 않아야 함.(All or Nothing)
    - 일관성(Consistency) :  실행 후 상태가 모순되지 않고 일관성이 유지되어야 한다.
    - 고립성(Isolation) : 연산의 중간결과는 다른 트랜잭션이 접근할 수 없다.
    - 영속성(Durability) : 실행 결과는 영구적으로 보장
- 실행 순서
    1. 파싱 : 문법 검사,구문 분석. 분석한 SQL문은 Live Cache에 저장 함.
    2. 실행 : Optimizer가 수립한 실행 계획에 따라 SQL 실행
    3. 인출 : 데이터를 읽어서 전송
   
### DDL(Data Definition Language)
#### Create Table
```sql
create table dept(
deptno varchar2(4) primary key,
deptname varchar2(20)
);

insert into dept values('1001', '인사팀');
insert into dept values('1002', '총무팀');

-- 테이블의 구조를 확인
DESC dept; 
```


#### CASCADE 
- 참조 관계가 있을 경우 참조되는 데이터를 자동으로 반영할 수 있는 것
- ON DELETE CASCADE : 자신이 참조하고 있는 데이터가 삭제되면 자신도 삭제되는 옵션. 참조 무결성 준수할 수 있다.
```sql
create table EMP(
    empno number(10), 
    ename varchar2(20),
    sal number(10,2) default 0, -- 소수점 둘째 자리까지 저장. mysql에서는 소수점 표현 위해 decimal,numeric 사용
    deptno varchar2(4) not null, -- 외래키와 매칭될 컬럼
    createdate date default sysdate, -- mysql에서는 NOW() 또는 CURRENT_TIMESTAMP
    constraint emppk primary key(empno), -- constraint로 기본키와 기본키 이름 지정.
    -- constraint emppk primary key(empno,ename) 두개의 컬럼을 기본키로 지정하는 경우
    constraint d_pk foreign key(deptno) references dept(deptno) on delete cascade -- 외래키 지정, cascade 옵션 사용
);

insert into EMP values(100,'kim',1000,'1001', sysdate);
insert into EMP values(101,'kim',2000,'1002', sysdate);

delete from dept where deptno='1001'; -- EMP의 deptno가 1001인 튜플이 삭제 됨
```

#### 테이블 변경
```sql
-- 테이블명 변경 : alter table ~ rename to
alter table emp rename to new_emp;

-- 칼럼 추가 : alter table ~ add
alter table emp add (age number(2) default 1);

-- 칼럼 변경 : alter table ~ modify
alter table emp modify (ename varchar2(40) not null);

-- 칼럼 삭제 : alter table ~ drop column
alter table emp drop column age;

-- 컬럼명 변경 : alter table ~ rename column ~ to
alter table emp rename column ename to new_ename;

-- 테이블 삭제 : drop table
-- drop table은 테이블 구조와 데이터 모두 삭제한다
drop table emp;
drop table emp cascade constraint; -- 해당 테이블의 데이터를 외래키로 참조한 슬레이브 테이블과 관련된 제약사항도 삭제할 때 사용
```

#### 뷰 생성과 삭제
- 뷰는 가상의 테이블로 데이터를 가지고 있지 않고 테이블을 참조해서 원하는 칼럼만을 조회할 수 있게 함.   
- 데이터 딕셔너리에 SQL문 형태로 저장하되 실행 시에 참조 됨.
- 특징
  - 보안성 향상, 데이터 관리가 간단하다.
  - 참조한 테이블이 변경되면 뷰도 변경 됨
  - 입력,수정,삭제에 제약이 있어 뷰를 변경하고자 하면 삭제 후 재생성해야 함.
```sql
-- 뷰 생성
create view t_emp as select * from emp;

-- 뷰 삭제
drop view t_emp;
```

- Nologging 옵션은 로그파일 기록 최소화 시켜 성능을 향상. Buffer Cache라는 메모리 영역을 생략하고 기록

### GROUP 연산
#### group by
- 소규모 행을 그룹화하여 합계,평균,최댓값,최솟값 등을 계산. 
- 그룹에 대한 조건문은 having구에 사용. where절에 조건 추가하면 group by 대상에서 제외 됨.
```sql
select deptno,sum(sal) from emp group by deptno having sum(sal) > 10000;

-- empno가 1000번에서 1003번인 사원들의 부서별 급여 합계
select deptno,sum(sal) from emp where empno between 1000 and 1003 group by deptno;
```

  WHEN condition1 THEN result_1

### ROWNUM과 ROUND
#### ROWNUM
- 오라클 DB에서 select문 결과에 대해서 논리적인 일련번호를 부여
- ROWNUM은 조회되는 행 수를 제한할 때 많이 사용
```sql
select * from emp where ROWNUM <= 1; -- 한 행만 조회

-- from 절에 사용되는 서브쿼리를 인라인뷰라고 함
-- ROWNUM에 별칭 list 부여. 5개 행만 조회
select * from (select ROWNUM list, ename from emp) where list <= 5;

select * from (select ROWNUM list, ename from emp) where list between 5 and 10;
```
#### SQL Server(MS-SQL)의 Top, MySQL의 limit
ROWNUM을 SQL server에서는 Top, MySQL에서는 limit 사용
```sql
-- MS-SQL
select top(10) from emp;

-- MySQL
select * from emp limit 10;
```

#### ROWID
- 오라클 DB 내에서 데이터를 구분할 수 있는 유일한 값
- ROWID를 통해 데이터가 어떤 데이터파일, 어느 블록에 저장되어 있는지 알 수 있음
```sql
select rowid, empno from emp;
-- AAASwGAABAAALtRAAB  |   101
```

### WITH
- 서브쿼리를 사용해서 임시 테이블이나 뷰처럼 사용할 수 있는 구문

### DCL
#### GRANT
- object table에 대한 privileges 권한을 user에게 부여한다.
    > GRANT privileges ON object TO user;
- 권한 : select, insert, update,delete,references,alter,index,all
- with grant option, with admin option
  - 권한을 부여할 수 있는 권한을 부여
  - with grant option : 권한을 A사용자가 B에 부여하고 B가 다시 C에게 부여한 후 revoke하면 모든 권한 히수 됨
  - with admin option : 권한을 A사용자가 B에 부여하고 B가 다시 C에게 부여한 뒤 revoke하면 B사용자 권한만 취소 됨
```sql
GRANT select,insert,update,delete ON EMP TO customer;

GRANT select,insert on emp to customer with grant option;

-- with admin option은 주로 역할(roles)에 대한 권한을 부여할 때 사용
-- 역할을 부여받은 사용자는 해당 역할에 연결된 권한을 다른 사용자에게도 부여할 수 있음
GRANT SELECT ON your_table TO DataViewer;
GRANT DataViewer TO user1 WITH ADMIN OPTION;
```

#### REVOKE
데이터베이스 사용자에게 부여된 권한을 회수
```sql
REVOKE privileges ON object FROM user;
```

### TCL(Transaction Control Language)
#### Commit
- commit이 완료되면 데이터베이스 변경으로 인한 Lock이 해제되며 하나의 트랜잭션 과정을 종료한다.
- DDL/DCL을 사용하는 경우 자동 commit됨. "set autocommit on;" 실행 시 자동 commit
#### Rollback
변경사항 모두 취소하고 트랜잭션 종료

#### savepoint
- 트랜잭션을 작게 분할하여 관리. 지정된 위치 이후의 트랜잭션만 rollback 가능
- savepoint 지정 : savepoint <SAVEPOINT명>
- 지정된 savepoint로 되돌리고 싶은 경우 : rollback to <SAVEPOINT명>
```sql
savepoint t1;
insert into emp values(10,20);
savepoint t2;
insert into emp values(20,30);
rollback to t2;
commmit;
select * from emp;
-- 10 20
```
