### JOIN
#### EQUI JOIN
EQUI JOIN은 "="을 사용해서 두 개의 테이블을 연결한다.
```sql
select * from emp, dept where emp.deptno = dept.deptno;
select * from emp, dept where emp.deptno = dept.deptno AND emp.ename like 'KIM%' order by ename;
```

#### INNER JOIN
EQUI JOIN과 같은 ISO 표준 SQL로 INNER JOIN 있음. ON문 사용해 연결
```sql
select * from emp inner join dept on emp.deptno = dept.deptno;
select * from emp inner join dept on emp.deptno = dept.deptno and emp.ename like 'KIM%' order by ename;
```
#### EQUI JOIN/INNER JOIN 실행 계획
- 테이블 전체를 읽은 다음(TABLE ACCESS FULL) 해시 함수를 사용해 연결
- 해시 조인은 EQUI JOIN/INNER JOIN에서만 사용 가능
- 해시 조인
  - 먼저 선행 테이블을 결정하고 선행 테이블에서 주어진 조건에 해당하는 행 선택
  - 해당 행이 선택되면 조인 키를 기준으로 해시 함수를 사용해서 해시 테이블을 메인 메모리에 생성하고 후행 테이블에서 주어진 조건에 만족하는 행 찾음
  - 후행 테이블의 조인 키를 사용해서 해시 함수를 적용하여 해당 버킷을 검색

#### INTERSECT 
두 개의 테이블에서 교집합 즉 공통된 값을 조회한다.
```sql
select deptno from emp intersect select deptno from dept;
-- deptno 10 20 30
```
#### Non-Equi JOIN
- "="을 사용하지 않고 ">","<",">=",">=" 등을 사용하여 조인.
- 즉 정확하게 일치하지 않는 것을 조인

#### OUTER JOIN
- 두 개의 테이블 간에 교집합을 조회하고 한쪽 테이블에만 있는 데이터도 포함
- 한쪽 테이블에만 있는 행을 포함하면 LEFT OUTER JOIN/RIGHT OUTER JOIN
- 양쪽 모두 포함하면 FULL OUTER JOIN
- Oracle에서는 outer join할 때 "(+)"를 사용 가능
```sql
-- left outer join  모두 포함하는 테이블의 다른 쪽 테이블에 (+) 붙여주다.
select * from dept,emp where emp.deptno (+)= dept.deptno;

-- 위와 같음
select * from dept left outer join emp on dept.deptno=emp.deptno;

-- right outer join
select * from dept,emp where emp.deptno = dept.deptno(+);
```

#### CROSS JOIN
조건구 없이 2개의 테이블을 조인하여 카테시안 곱 발생
```sql
select * from emp cross join dept;
```

#### UNION
- 두 개의 테이블을 하나로 합친다. 
- 이 때 두 개의 테이블의 칼럼 수, 데이터 형식이 일치해야 함.
- 중복된 데이터를 제거한다. 그래서 정렬이 발생한다.

#### UNION ALL
- 두개의 테이블을 합치며 중복 제거하지 않는다. 정렬이 발생하지 않음

#### MINUS
- 차집합 조회. MS-SQL에서는 EXCEPT
```sql
select deptno from dept minus select deptno from emp;
```

### 계층형 조회(CONNECT BY)
- Oracle DB에서 지원하는 계층형 조회는 트리 형태의 구조를 위에서 아래,또는 역으로 탐색하면서 조회하는 것
- CONNECT BY : 트리 구조로 질의 수행
- CONNECT BY 키워드   
  |키워드|설명|
  |--|--|
  |LEVEL|검색 항목의 깊이를 의미. ROOT는 1|
  |CONNECT_BY_ROOT|ROOT값을 표시|
  |CONNECT_BY_ISLEAF|리프노트인지 1,0으로 표시|
  |SYS_CONNECT_BY_PATH|전체 전개 경로 표시|
  |NOCYCLE|순환 구조가 발생지점까지만 전개, 발생한 지점까지 허용하고 오류 없이 진행 됨|
  |CONNECT_BY_ISCYCLE|순환구조의 일부분인지 표시|
- 계층형 조회 키워드   
  |키워드|설명|
  |--|--|
  |START WITH 조건|시작 위치 지정|
  |PRIOR|바로 직전에 출력된 행|
  |PRIOR 자식=부모|부모에서 자식방향으로 순방향 전개|
  |PRIOR 부모=자식|자식에서 부모방향으로 역방향 전개|
  |NOCYCLE|사이클이 발생되지 않게 함|
  |Order sibilings by 컬럼명|동일한 레벨인 형제 노드 사이에 정렬 수행|

- 예제
  - mgr 1000번은 1001,1002번 관리. 1002는 1009,1010을 관리     
    |empno|mgr|ename|
    |--|--|--|
    |1000||test1|
    |1001|1000|test2|
    |1002|1000|test3|
    |1009|1002|test4|
    |1010|1002|test5|

    ```sql
    - MAX(LEVEL) : 최대 계층 수, 계층 형 구조에서 마지막 Leaf Node의 계층값
    - start with mgr is null : 1000이 Root가 됨, Root는 레벨1
    select MAX(LEVEL) from emp start with mgr is null connect by prior empno=mgr;
    -- MAX(LEVEL) 3 : 트리의 최대 깊이는 3


    - LPAD : 트리 형태 보기 위해 사용. 왼쪽에 지정된 문자를 숫자만큼 채워넣음. 
    -- LPAD(' ', 3*(LEVEL-1)) 공백을 레벨-1만큼 채워서 출력
    select LEVEL, LPAD(' ',4*(LEVEL-1))||empno,mgr,CONNECT_BY_ISLEAF ISLEAF
    from emp 
    start with mgr is null
    connect by prior empno=mgr;

    -- empno 1010가 ROOT가 됨. 
    -- emp 테이블을 탐색하며 현재행의 mgr이 상위행의 empno와 같으면 연결
    -- 역방향 전개가 됨 자식->부모
    SELECT LEVEL, LPAD(' ',3*(LEVEL-1))||empno, mgr, CONNECT_BY_ISLEAF ISLEAF
    from emp
    start with empno=1010
    connect by prior mgr=empno;
    ```

### 서브쿼리
- 인라인 뷰 : from구에 select문 사용
- 스칼라 서브쿼리 : select문에 subquery 사용
- 서브쿼리 : where구에 select문 사용
```sql
select *
from emp
where deptno = (select deptno from dept where dept=10); -- subquery
```
#### 단일행 서브쿼리와 다중행 서브쿼리
|||
|--|--|
|단일행 서브쿼리|결과는 반드시 한 행, 비교연산자 =,<,>,>=,<=,<> 사용|
|다중행 서브쿼리|여러개의 행 반환. 다중 행 비교연산자 IN, ANY, ALL, EXISTS 사용|

#### 다중행 서브쿼리
- IN
```sql
select ename,sal from emp,dept
where emp.deptno=dept.deptno
and emp.empno in (select empno from emp where sal > 2000);
```
- ALL
```sql
-- deptno가 20,30 모두보다 작거나 같으면 조회 됨
select * from emp
where deptno <= ALL(20,30);
-- deptno 20 20 10 20 20 20
```
- EXISTS
데이터가 존재 하면 참 아닌 경우 거짓
```sql
-- sal이 2000보다 큰 데이터가 존재하면 exists에 의해 true가 됨
-- true인 경우 즉 sal이 2000보다 큰 데이터가 존재하는 경우만 조인결과 출력됨
-- 아닌 경우 출력 안됨
select ename,sal from emp,dept
where emp.deptno=dept.deptno
and exists (select 1 from emp where sal>2000);
```

#### 스칼라 서브쿼리
select에 사용하며 반드시 한 행과 한 칼럼만 반환
```sql
select ename as "name", (select avg(sal) from emp) AS "AVG"
from emp where empno=1000;
-- KIM 203.45 ; name, avg 출력 됨
```
#### 연관(Correlated) Subquery
서브쿼리 내에서 메인쿼리 내의 컬럼을 사용
```sql
select * from emp a
where a.deptno = (select deptno from dept b where b.deptno=a.deptno);
```

### 그룹 함수
#### ROLLUP
- GROUP BY 칼럼에 대해 Subtoal을 만들어준다.
- GROUP BY구에 칼럼이 두 개 이상 오면 순서에 따라서 결과가 달라진다.
```sql
select decode(deptno, null,'total', deptno), sum(sal) from emp
group by rollup(deptno);
부서별 합계 출력 후 roolup에 의해 전체 합계가 출력되며 null이므로 decode에 의해 total 문구 출력 됨
-- 마지막 행에 total 30000  : 전체 합계가 조회된다.

select deptno,job,sum(sal)
from emp
group by rollup(deptno,job);
부서별,직업별 두개 이상 ROLLUP 실행하면 처음에 DEPTNO와 JOB으로 그룹화를 한 후, DEPTNO만으로 그룹화하고, 마지막으로 모든 행에 대한 전체 합계를 계산
/*
10 CLERK 1000
10 MANAGER 2000
10         3000
20 CLEARK 3000
20 ANALIST 4000
20         7000
           10000
*/
```
#### Grouping 함수
- ROLLUP, CUBE, GROUPING SET에서 생성되는 합계값을 구분하기 위해 만들어진 함수
- 소계,합계 등이 계산되면 GROUPING 함수는 1 반환, 아니면 0 반환
```sql
select deptno,grouping(deptno),job,grouping(job), sum(sal)
from emp
group by rollup(deptno, job);
/*
deptno | grouping(deptno)| job    | grouping(job)| sum(sal)
10     |         0       | CLERK  |   0          | 1000
10     |         0       | MANAGER| 0            | 2000  
10     |         0       |        |  1           | 3000
20     |         0       | CLEARK |     0        | 3000
20     |         0       | ANALIST|      0       | 4000
20     |         0       |        |     1        | 7000
       |         1       |        |      1       |10000
*/

select deptno, decode(grouping(deptno,1,'전체합계') tot,job,
      decode(grouping(job),1,'부서합계') t_dept, sum(sal)
from emp
group by rollup(deptno, job);
-- decode로 grouping이 1일 때 합계 문구 표시
```

#### GROUPING SETS
- group by에 나오는 칼럼의 순서와 관계 없이 개별적으로 모두 소계를 처리
```sql
select deptno,job,sum(sal)
from emp
group by grouping sets(deptno, job);

직업,부서별 합계가 개벽적으로 조회 됨
/*
deptno | job     | sum(sal)|
       | CLERK   | 3000    |
       | SALESMAN| 3000    | 
10     |         | 4000    |
20     |         | 4000    |
*/
```

#### CUBE
- 제시된 칼럼에 대해 결합 가능한 모든 집계를 계산
- 예를 들어 부서와 직업을 CUBE로 사용하면 부서별,직업별,부서별 직업별, 전체합계가 조회 됨
```sql
select deptno,job,sum(sal)
from emp
group by cube(deptno, job);
```
