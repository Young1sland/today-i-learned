kafka 다운
 - bin 폴더 : 실행파일
 - config 폴더 : 환경 설정 파일
 - kafka를 실제로 쓸 때는 zookeeper, kafka 각 3개 이상 설정하는게 안정적이다.

Kafka 실행
zookeeper 실행
./bin/zookeeper-server-start.sh ./config/zookeeper.properties

Kafka 실행
./bin/kafka-server-start.sh ./config/server.properties

Topic 확인
./bin/kafka-topics.sh --bootstrap-server localhost:9092 --list

Topic 생성
./bin/kafka-topics.sh --bootstrap-server localhost:9092 \
 --create --topic {topic 이름} --partions 1

Topic 목록 확인
./bin/kafka-topics.sh --bootstrap-server localhost:902 --list

자세한 Topic 정보 조회
./bin/kafka-topics.sh --bootstrap-server localhost:9092 \
 --describe --topic {topic 이름}

* 터미널에서 동작 확인해 볼 수 있도록 제공되고 있다.
producer 등록
./bin/kafka-console-producer.sh --broker-list localhost:9092 --top
ic {topic 이름}

consumer 등록
./bin/kafka-console-consumer.sh --bootstrap-server localhost:9092 --topic 
 {topic 이름} --from-beginning
* --from-beginning 처음부터 끝까지 다 가져오기

kafka source connect -> kafka cluster -> kafka Sink connect
 Source Connect가 데이터를 전달하면 Topic에 쌓임. 
 Sink Connect는 Topic에 있는 데이터를 받아 처리.

Kafka Sink Connect test
  - Sink Connect 추가
    {
      "name": "my-sink-connect",
      "config":{
          "connector.class" : "io.confluent.connect.jdbc.JdbcSinkConnector",
          "connection.url" : "jdbc:mysql://localhost:3306/mydb",
          "connection.user" : "root",
          "connection.password" : "chunjae4",
          "auto.create" : "true", //Topic의 이름과 같은 테이블 생성
          "auto.evolve" : "true", 
          "delete.enabled": "false",
          "tasks.max" : "1",
          "topics":"my_topic_users"
      }
    }
  - 등록하고 db 확인하면 my_topic_users table이 생성되어 있음.
    user 디비에 insert하면 my_topic_users 테이블도 insert됨
