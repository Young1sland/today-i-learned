# Redis
- In-memory 데이터베이스로 key-value 형태의 데이터베이스  
  *In-memory이므로 휘발성. 꺼지면 데이터 사라짐.  
  https://redis.io/    
- Single thread
- 높은 availability를 위한 몇가지 옵션 존재
  - Sentinel : In-memory이므로 데이터 유실될 경우 고려하여 여러 개의 redis를 하나의 instance처럼 사용.
  - Redis Cluster
```
기본 SET,GET
redis> SET wisdom "genius"
redis> GET wisdom
redis> SET alright "okok" EX 60 (60초 뒤 Expired)
```

```
HKEYS
Syntax : HSET key field value
redis> HSET myhash field1 "Hello"
redis> HSET myhash field2 "World"
값 가져오기
redis> HGET myhash field1
    "Hello"
key로 조회
redis> HKEYS myhash
1) "field1"
2) "field2"
삭제. field 단위로 지움.
redis> HDEL myhash field1
    (Integer)1
```

```
ZADD
Syntax : ZADD key [NX | XX] [GT | LT] [CH] [INCR] score member
redis> ZADD myzset 1 "one"  //"one" member에 score 1 지정
(integer) 1
redis> ZADD myzset 1 "uno"
(integer) 1
redis> ZADD myzset 2 "two" 3 "three"
(integer) 2
redis> ZRANGE myzset 0 -1 WITHSCORES //스코어랑 같이 0부터 -1(전체)까지 가져오기
1) "one"
2) "1"
3) "uno"
4) "1"
5) "two"
6) "2"
7) "three"
8) "3"
redis> ZRANGE myzset 0 2 //0~2 score에 해당하는 member 가져오기
1) "one"
2) "uno"
3) "two"
```
