알람리스트를 가져오는 경우를 예로 들면 사용자 인증을 위해 User Entity에 접근하고 userEntity를 가져오기 위해 또 query가 발생한다.

```java
//Object를 class로 안전하게 casting하기 위한 function
 public static <T> Optional<T> getSafeCastInstance(Object o, Class<T> clazz){
        /*
        Class isInstance(Object o) :
            returns true if the specified Object argument is non-null and can be cast to the
            reference type represented by this Class object without raising a ClassCastException.
        */
        return clazz != null && clazz.isInstance(o) ? Optional.of(clazz.cast(o)) : Optional.empty();
    }

//UserEntity 다시 가져오는 대신 anthentication의 user 사용
@GetMapping("/alarm")
public Response<Page<AlarmResponse>> alarm(Pageable pageable, Authentication authentication){
    User user = ClassUtils.getSafeCastInstance( authentication.getPrincipal(), User.class).orElseThrow(()-> new SnsApplicationException(ErrorCode.INTERNAL_SERVER_ERROR, "Casting to User class failed"));
    return Response.success(userService.alarmList(user.getId(), pageable).map(AlarmResponse::fromAlarm));
}

//userId로 query
 public Page<Alarm> alarmList(Integer userId, Pageable pageable){
        return alarmEntityRepository.findAllByUserId(userId, pageable).map(Alarm::fromEntity);
}
```

## N+1 문제 해결  
Entity에 Relation을 정의해 두고 SELECT했을 때 한번의 쿼리가 발생할 것을 기대하지만 실제로는 연관된 Entity 개수만큼 쿼리가 더 발생하는 문제가 있다.


### fetch type 변경
default인 EAGER 타입은 해당 릴레이션의 Entity의 사용여부와 상관없이 무조건 가져오는 쿼리가 발생한다.  
LAZY로 바꿔주면 사용하지 않는 경우 쿼리하지 않으므로 불필요한 쿼리를 줄일 수 있다. 그러나 N+1문제의 근본 해결책은 아님.
```java
public class AlarmEntity {
    ...
    // default EAGER
    @ManyToOne(fetch = FetchType.LAZY)
    @JoinColumn(name = "user_id")
    private UserEntity user;
    ...
}
```
N+1문제의 근본 해결책은 @Query(..)를 사용하여 쿼리를 사용하여 조회하는 방법을 사용하는 것임.

### POST 삭제 시 comment,like도 삭제하기
JPA에서 제공하는 deleteAllByPost를 사용해서 삭제할 수 있다.  
그러나 이 경우 상당히 비효율적으로 동작하므로 @Query로 만들어서 사용하는 것이 좋다.  
예를 들면 Post에 연관된 comment를 모두 삭제한다고 하면 JPA에서 제공하는 deleteAll을 사용할 경우 comment에 하나하나 deleteAt을 업데이트하여 쿼리가 여러번 발생하게 된다.

```java
@Repository
public interface CommentEntityRepository extends JpaRepository<CommentEntity, Integer> {
    ..
    //delete는 만들어 사용하자. delete의 경우 @Modifying 필요
    @Transactional
    @Modifying
    @Query("UPDATE CommentEntity entity SET deleted_at = NOW() where entity.post = :post")
    void deleteAllByPost(@Param("post")PostEntity post);

    //void deleteAllByPost(PostEntity post);
}

 public void delete(String userName, Integer postId){
    PostEntity postEntity = getPostEntityOrException(postId);
    UserEntity userEntity = getUserEntityOrException(userName);

    if(postEntity.getUser() != userEntity){
        throw new SnsApplicationException(ErrorCode.INVALID_PERMISSION,
                String.format("%s has no permission  with %s", userName, postId));
    }
    likeEntityRepository.deleteAllByPost(postEntity);
    commentEntityRepository.deleteAllByPost(postEntity);
    postEntityRepository.delete(postEntity);
}
```
