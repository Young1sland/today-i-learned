### Spock 프레임워크
- Groovy 언어를 이용하여 테스트 코드를 작성할 수 있는 프레임워크
- JUnit과 비교하여 코드를 더 간결하게 작성 가능
- Groovy언어는 동적 타입 프로그래밍 언어. JVM 위에서 동작하며 Java 문법과 유사
- 테스트 메소드 이름을 문자열로 작성할 수 있음. given, when, then 코드 블록을 명확히 구분

### Spock 작성 방법
- 테스트 클래스는 Groovy 클래스로 생성, Specification 클래스 상속 받음
- feature def를 이용하여 함수 선언. 하나 이상 블록이 존재해야 함.
- given 블록 : 테스트에 필요한 값 준비
- when 블록 : 테스트할 코드 실행
- then 블록 : when과 함께 사용하며 예외 및 결과 값을 검증 함
- expect 블록 : then과 같으며 when을 필요로 하지 않기 때문에 간단한 테스트 및 where와 같이 사용 됨.
- where 블록 : 데이터가 다르고 로직이 동일한 경우 동일한 테스트에 대한 중복 제거 가능
- https://spockframework.org/spock/docs/1.3/all_in_one.html
```groovy
class Example extends Specification {
    def setupSpec(){
        //첫 메소드 전 실행
    }
    def setup(){
        //모든 메소드 실행 전
    }
    def cleanup(){
        //모든 메소드 실행 후
    }
    def cleanpupSpec(){
        //마지막 메소드 실행 후
    }
}
```
- gradle에 플러그인 및 의존성 추가
```gradle
plugins {
    id 'java'
    id 'groovy'
}

testImplementation 'org.spockframework:spock-core:2.1-groovy-3.0'
testImplementation 'org.spockframework:spock-spring:2.1-groovy-3.0'

//런타임에 클래스 기반 spock mock 만들기 위해 필요
testImplementation 'net.bytebuddy:byte-buddy:1.12.10'
```
