### 영속성 컨텍스트
어플리케이션과 데이터베이스 사이에 entity를 보관하는 가상의 데이터베이스 같은 역할
1. 1차 캐시, 쓰기 지연 SQL 저장소
   - persist하는 순간 pk값, 타입과 객체를 맵핑하여 1차 캐시 넣고 쓰기 지연 SQL 저장소에 쿼리를 만들어 쌓음(단,자동 생성되는 id값과 같이 pk값을 알 수 없는 경우는 먼저 쓰고 캐시에 저장)
   - 한 트랙잭션 내에 1차 캐시에 이미 있는 값을 조회하는 경우 DB를 조회 하지 않고 1차 캐시에 있는 내용 그대로 가져옴
   - 단 1차 캐시는 어플리케이션 전체 공유가 아닌 한 트랜잭션 내에서만 공유
   - 조회 했을 때 1차 캐시에 없다면 DB에서 가져와서 1차 캐시에 저장 후 반환
   - commit하는 순간 영속성 컨텍스트의 변경 내용을 DB에 반영(flush)
2. JPA Dirty Checking
   - Dirty Checking이란 entity의 상태 변경 검사. Dirty란 상태의 변화가 생긴 정도를 의미
   - JPA에서 트랜잭션이 끝나는 시점에 변화가 있는 모든 entity 객체를 데이터베이스에 자동으로 반영해 준다.
   - 1차 캐시에 처음 들어온 상태인 Entity 스냅샷을 넣어둠.
   - flush() 호출되면 엔티티와 스냅샷 비교하여 변경된 값이 있으면 update 쿼리를 쓰기 지연 SQL에 넣어두고 DB에 반영한다.
