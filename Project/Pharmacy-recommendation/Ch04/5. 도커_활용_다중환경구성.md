- docker compose 환경 파일 경로에 .env 파일울 구성하면 별다른 설정 없이 Docker Compose에 바로 반영
- 비밀정보가 코드에 노출되지 않도록 .gitignore에 .env 파일 추가
```shell
$ docker-compose config #.env파일 자동 반영
$ docker-compose up     # 이미지 없을 때 이미지 빌드 후 컨테이너 실행
$ docker-compose up --build # 이미지 있든 없든 재빌드하여 컨테이너 실행
$ docker-compose -f docker-compose-local.yml up
```
- 추후 암호화를 통해 더 높은 보안을 제공하는 오픈소스 HashiCorp Vault 고려

### database 설정
- docker-compose로 실행할 경우 .env에 정의해 둔 환경설정 사용할 수 있지만 로컬로 실행할 경우 intelliJ Run Configuration에 넣어주자.
``` Run Configuration-> Modify options->Environment variables```
```yml
spring:
  config:
    activate:
      on-profile: local
  datasource:
    driver-class-name: org.mariadb.jdbc.Driver
    url: jdbc:mariadb://localhost:3306/pharmacy-recommendation
    username: ${SPRING_DATASOURCE_USERNAME}
    password: ${SPRING_DATASOURCE_PASSWORD}
```
