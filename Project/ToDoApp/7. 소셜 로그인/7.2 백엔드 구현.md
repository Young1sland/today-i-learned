### 소셜 로그인
- 싱글 사인 온(Single Sign On) : 하나의 아이디로 여러개의 독립된 App에 로그인할 수 있는 인증 매커니즘
- 소셜 로그인은 싱글 사인 온의 일종이라 할 수 있고 구현 방법 중 하나가 OAuth2임

### 소셜 로그인(Github) 사이트에서 클라이언트 App 생성하기
- 리소스 서버 및 인가 서버는 클라이언트 App을 알고 있어야 함.
- github-> Settings -> Developer settings -> OAuth Apps->Register
- Homepage URL : http://localhost:8080 (Backend URL)
- Authorization callback URL : http://localhost:8080/oauth2/callback (Backend URL의 Callback endpoint)
- Client ID, Secret Key 기록

### application-dev.yaml에 OAuth2.0 설정
설정된 프로퍼티들은 이후 스프링 시큐리티 OAuth2.0 라이브러리에 의해 ClientRegistration.java 형태로 변경 된다.
```yaml
spring:
  security:
    oauth2:
      client:
        registration:
          <RegistrationId>: # 어떤 리소스 제공자의 클라이언트인지 ex) github
            clientId: <ClientId>
            clientSecret: <Client Secret>
            redirectUri: "{baseUrl}/oauth2/callback/{registrationId}"
            scope: # 어떤 리소스 필요한지
              - <SCOPE 1>
              - <SCOPE 2>
    provider: #리소스 제공자, 소셜 로그인 요청하면 authorization uri로 리다이렉트 됨
      github:
        authorization-uri: https://github.com/login/oauth/authorize
```

### Todo 백엔드 OAuth2.0 엔드포인트 설정
- gradle에 oauth2.0 라이브러리 추가
```gradle
	implementation 'org.springframework.security:spring-security-oauth2-client:5.6.1'
```
```java
public class WebSecurityConfig extends WebSecurityConfigurerAdapter {
  ...
    @Override
    protected void configure(HttpSecurity http) throws Exception {
        http
        ...
            .antMatchers("/", "/auth/**","/oauth2/**").permitAll() // /,/auth/** 인증 X
            .anyRequest().authenticated()
            .and()
            .oauth2Login(); //oauth2Login 설정
    }
```
- 애플리케이션 실행 후 http://localhost:8080/oauth2/authorization/github로 들어가면 깃허브 로그인 화면으로 리다이렉트 됨
```shell
$ ./gradlew bootRun --args='--spring.profiles.active=dev'
```
- 인가 되면 http://localhost:8080/oauth2/callback/github?XX로 리다이렉트 되며 페이지 존재하지 않아 HTTP 404 반환
- redirectEndPoint 설정
    - .baseUri("/oauth2/callback/*")로 들어오는 요청을 redirectionEndPoint()로 리다이렉트.
    - redirectEndPoint에 아무 설정하지 않은 경우 베이스 URI인 http://localhost:8080으로 리다이렉트    
```java
public class WebSecurityConfig extends WebSecurityConfigurerAdapter {
    ...
    @Override
    protected void configure(HttpSecurity http) throws Exception {
        http.
        ...
            .oauth2Login() //oauth2Login 설정
            .redirectionEndpoint()
            .baseUri("/oauth2/callback/*");
    }
}
```

### 소셜 로그인 후 자동 회원 가입
- token-uri, user-info-uri 주소 추가
```yaml
   provider:
      github:
        authorization-uri: https://github.com/login/oauth/authorize
        token-uri: https://github.com/login/oauth/access_token
        user-info-uri: https://api.github.com/user
```
- 유저 정보 가져오기 위해서는 액세스 토큰이 필요하므로 우리는 token-uri를 이용해 먼저 액세스 토큰을 받은 후 user-info-uri로 사용자의 정보를 요청할 때 토큰 함께 보냄
- 이런 작업을 대신 해주는 클래스가 DefaultAuthrizationCodeTokenResponseClient, DefaultOAuth2UserService
- DefaultOAuth2UserService를 extends 하여 OAuthUserServiceImple.java 작성
- WebSecurityConfig에 userInfoEndpoint와 userService 추가
```java
public class WebSecurityConfig extends WebSecurityConfigurerAdapter {
@Autowired
private OAuthUserServiceImpl oAuthUserService;
 @Override
    protected void configure(HttpSecurity http) throws Exception {
        http.
            .userInfoEndpoint()
            .userService(oAuthUserService); //OAuthUserServiceImpl을 유저 서비스로 등록        
    }
}
```

### Todo App 토큰 발행
- 토큰 발행은 OAuth2.0 인증이 완료된 다음 발행해야 함.
- OAuthSuccessHandler 내부에서 토큰 생성하고 반환
- OAuthSuccessHandler 구현하고 WebSecurityConfig에 추가
```java
//주의 onAuthenticationSuccess의 매개변수가 맞는 메소드를 오버라이드
public class OAuthSuccessHandler extends SimpleUrlAuthenticationSuccessHandler {
  @Override
    public void onAuthenticationSuccess(HttpServletRequest request, HttpServletResponse response, Authentication authentication) throws IOException, ServletException {
        TokenProvider tokenProvider = new TokenProvider();
        String token = tokenProvider.create(authentication);
        response.getWriter().write(token);
        log.info("token {}", token);
    }
}

public class WebSecurityConfig extends WebSecurityConfigurerAdapter {   
    @Autowired
    private OAuthUserServiceImpl oAuthUserService;

    @Autowired
    private OAuthSuccessHandler oAuthSuccessHandler; //Success Handler

    @Override
    protected void configure(HttpSecurity http) throws Exception {
        http.
        ...
            .oauth2Login() //oauth2Login 설정
            .redirectionEndpoint()
            .baseUri("/oauth2/callback/*")
            .and()
            .userInfoEndpoint()
            .userService(oAuthUserService) //OAuthUserServiceImpl을 유저 서비스로 등록
            .and()
            .successHandler(oAuthSuccessHandler);      
    }
}
```
