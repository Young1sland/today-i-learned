### 백엔드 AuthenticationEntryPoint 설정
- 스프링 시큐리티 기본 설정으로 인해 토큰 없어 인증 실패한 경우 403을 반환하지 않고 OAuth2.0 흐름에 따라 리다이렉트 됨.
- AuthenticationEntryPoint를 이용해 인증에 실패한 요청의 응답을 원하는 대로 정할 수 있음
- AuthenticationEntryPoint는 인증되지 않은 사용자를 처리하기 위한 시작점이란 의미로 이해하자.
- 새 클래스 만들어 지정할 수도 있지만 403 반환하기만 하면 되므로 Http403ForbiddenEntryPoint를 사용

```java
 protected void configure(HttpSecurity http) throws Exception {
        http...
            .and()
            .exceptionHandling()
            .authenticationEntryPoint(new Http403ForbiddenEntryPoint());
 }
```

### 소셜 로그인 엔드포인트 추가
- OAuth 진입 경로는 원하는대로 설정 가능    
  기존 테스트에 사용하던 http://localhost:8080/oauth2/authorization/github가 아닌 auth/authorize/{provider}로 변경해 본다.
```java
    @Override
    protected void configure(HttpSecurity http) throws Exception {
        http...
            .and()
            .authorizationEndpoint()
            .baseUri("/auth/authorize")
```
- http://localhost:8080/auth/authorize/github로 OAuth 진입 됨
