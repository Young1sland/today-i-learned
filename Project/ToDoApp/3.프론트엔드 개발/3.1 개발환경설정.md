### react project 시작
```shell
$ npx create-react-app todo-react-app
$ npm start

# package-lock.json은 패키지의 버젼을 고정하기 위한 파일
# node_modules, package-lock.json을 삭제하면 새 버젼을 받아 설치
$ rm -rf node_modules
$ rm package-lock.json
$ npm i

# material-ui 사용
# https://material-ui.com
$ npm i @mui/material@5.6.0 @mui/icons-material@5.6.0
$ npm i @emotion/react@11.9.0 @emotion/styled@11.8.1
```

### HTML이 브라우저에 보여지는 단계
1. 파싱
   - 브라우저는 HTML을 트리 자료구조의 형태인 DOM(Document Object Model)트리로 변환
   - image,css,script 리소스 다운로드
   - CSS의 경우 다운로드하고 CSSOM(CSS Object Model)로 변환
   - 자바스크립트를 인터프리트,컴파일,파싱,실행
2. 렌더링
   - DOM,CSSOM 트리 합쳐 렌더 트리 만듬
   - 레이아웃 정함. 트리의 각 노드가 브라우저의 어디에 배치될지와 크기 정함
   - 렌더트리의 각 노드를 그려줌

### SPA(Single Page Application)
- 한번 웹페이지를 로딩합면 유저가 임의로 새로고침하지 않는 이상 페이지를 새로 로딩하지 않는 애플리케이션을 의미. JS가 동적으로 HTML 재구성해 만듬.   
- 클라이언트-사이드 렌더링 : 서버에게 새 HTML페이지 요청하지 않고 JS가 동적으로 HTML 재구성해 만드는 렌더링 과정
