### 일래스틱 빈스톡 애플리케이션 생성
- 로컬 환경에서 일래스틱 빈스톡 애플리케이션 초기화. 실제 AWS에는 아무 작업 하지 않음.
- .elasticbeanstalk/config.yml에 설정 저장 됨.
```shell
$ eb init TodoApp # project 이름이랑 일치해야 되는 듯
```

### 백엔드 애플리케이션 설정
- 애플리케이션 프로퍼티 파일을 개발용(dev)과 배포용(prod)으로 분리
- application-dev.yaml은 비워두면 기본으로 h2 DB 사용
```yaml
# application-prod.yaml
spring:
  jpa:
    database: MYSQL
    show-sql: true
    database-platform: org.hibernate.dialect.MySQL8Dialect
    hibernate:
      ddl-auto: update
    datasource:
      url: jdbc:mysql://${rds.hostname}:${rds.port}/${rds.db.name}
      username: ${rds.username}
      password: ${rds.password}
```
- MySQL8Dialect : 자바의 데이터형과 데이터베이스의 데이터형을 매핑해주는 라이브러리    
  String->Varchar, @Id 등의 어노테이션을 데이터베이스 키워드로 전환
- hibername.ddl-auto : 애플리케이션 시작 시 데이터베이스 테이블 설정    
    create : 테이블 삭제 후 새로 생성    
    update : 없는 테이블은 새로 생성. 수정된 테이블은 업데이트    
    validate : DB 테이블 스키마가 앱에 정의된 모델과 일치하는지 확인만 함.    
- ddl(Data Definition Language) : create, alter, drop 등의 쿼리 의미

### target jar 설정
일래스틱 빈스톡에게 애플리케이션을 자동으로 S3에 업로드하고 일래스틱 빈스톡 환경에 설정해 달라고 알려줘야 함.
```shell
$ ./gradlew build
# build/libs 아래에 jar 생성됨.
```
```yml
# elasticbeanstalk.config.yml에 deploy 추가
deploy:
  artifact: build/libs/todoapp-0.0.1-SNAPSHOT.jar
```

### eb create로 AWS에 환경 생성
- eb create : AWS에 일래스틱 빈스톡 환경 생성
- --database : 생성하는 환경에 RDS 데이터베이스 만들기 위한 매개변수. 자동으로 데이터 베이스 생성 됨
- --elb-type : 일래스틱 로드 밸런서 타입 매개변수.    
               application, classic, network 타입이 있음.    
               https://aws.amazon.com/ko/elasticloadbalancing/features/?nc=sn&loc=2
- --instance-type : 인스턴스 타입
```shell
$ eb create --database --elb-type application --instance-type t3.micro
```

### 스프링 애플리케이션 프로파일 설정
eb deploy로 배포 전 스프링 애플리케이션 프로파일 설정
```shell
# prod로 설정. application-prod.yaml 사용
$ eb setenv SPRING_PROFILES_ACTIVE=prod
# dev로 설정하면 application-dev.yaml 사용
```

### Gradle 통한 배포 자동화
- 위의 과정 포함하여 gradle 통한 배포 자동화. 빌드 후 setenv, jar 배포 자동화
- build.gradle에 Task를 추가한다. *build,bootrun 등이 gradle task
```gradle
// build.gradle
task('release'){
	//gradlew build 먼저 실행
	dependsOn("build")
	doLast {
		def stdout = new ByteArrayOutputStream()
		//exec
		exec {
			commandLine 'eb','setenv','SPRING_PROFILES_ACTIVE=prod'
			standardOutput=stdout
		}
		//logging result
		println "eb setenv SPRING_PROFILES_ACTIVE=prod :\n$stdout";
		exec {
			//eb deploy
			commandLine 'eb','deploy'
			standardOutput = stdout
		}
		println "eb deploy :\n$stdout";
		println "Release succeeded."
	}
}
```
```shell
$ ./gradlew release
```

### 환경 구성
- Elastic Beanstalk console에서 확인 가능
- Elastic Beanstalk은 태그에 환경을 붙여 구별하고 관리함
- CloudFormation의 템플릿->Designer 보기 통해 리소스 개괄적으로 확인 가능
- RDS CLI로 확인
```shell
$ aws rds describe-db-instances --region ap-northeast-2
```
- ASG(Auto Scaling Group) CLI로 확인
```shell
$ aws autoscaling describe-auto-scaling-groups --region ap-northeast-2
```
- 로드 밸런서 확인
```shell
$ aws elbv2 describe-load-balancers --region ap-northeast-2
```
- 로드 밸런서, ASG 연결하는 target group 확인
```shell
$ aws elbv2 describe-target-groups --region ap-northeast-2
```
- CloudWatch
```shell
$ aws ec2 describe-instances --region ap-northeast-2
```
