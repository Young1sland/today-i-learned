## DBMS 종류와 버젼 알아내기
- 버젼 조회
    ```sql
    -- 버젼 조회
    select @@version;
    select version;
    +-----------+
    | version() |
    +-----------+
    | 8.1.0     |
    +-----------+
    ```

- 에러 메시지 출력
  ```sql
  select 1 union select 1, 2;
  # MySQL => ERROR 1222 (21000): The used SELECT statements have a different number of columns
  select * from not_exists_table;
  # SQLite => Error: no such table: not_exists_table

  -- ERROR 1222는 MySQL에서 명시한 코드로 DBMS가 MySQL임을 알수 있다.
  mysql> select 1 union select 1, 2;
  ERROR 1222 (21000): The used SELECT statements have a different number of columns

  -- Postgres Error 
  select 1 union select 1, 2;
  ERROR:  each UNION query must have the same number of columns
  LINE 1: select 1 union select 1, 2;

  -- MSSQL
  select 1 union select 1, 2;
  Msg 205, Level 16, State 1, Server e2cb36ec2593, Line 1
  All queries combined using a UNION, INTERSECT or EXCEPT operator must have an equal number of expressions in their target lists.asdf

  -- SQLLite
  sqlite> select 1 union select 1, 2;
  Error: SELECTs to the left and right of UNION do not have the same number of result columns
  ```

- 참 또는 거짓 출력만 가능한 경우
  ```sql
  mid(@@version, 1, 1)='5'; -- substr과 같다고 보면 됨. mysql만 가능
  substr(version(), 1, 1)='P';

    -- Mysql @@version => '5.7.29-0ubuntu0.16.04.', mid(@@version, 1, 1) => '5'
    mysql> select mid(@@version, 1, 1)='5';
    +------------------------+
    | mid(@@version,1,1)='5' |
    +------------------------+
    |                      1 |
    +------------------------+

    -- Postgres
    select substr(version(), 1, 1)='P';
    ?column?
    ----------
    t
    (1 row)

    -- @@version => 'Microsoft SQL Server...', substring(@@version, 1, 1) => 'M'
    > select 1 from test where substring(@@version, 1, 1)='M';
    -----------
            1
    (1 rows affected)

    -- sqlite_version() => '3.11.0', substr(sqlite_version(), 1, 1) => '3'
    sqlite> select substr(sqlite_version(), 1, 1)='3';
    1
  ```
- 예외 상황
  - 어떤한 출력도 하지 않는 경우 시간 지연함수를 사용
  ```sql
    sleep(10)
    pg_sleep(10)

    -- mid 첫번째 문자가 일치하면 sleep(2)가 동작하여 시간지연이 발생. 아닌 경우 sleep(2) 동작하지 않음
    select mid(@@version, 1, 1)='6' and sleep(2);
    +---------------------------------------+
    | mid(@@version, 1, 1)='6' and sleep(2) |
    +---------------------------------------+
    |                                     0 |
    +---------------------------------------+
    1 row in set (0.00 sec)

    mysql> select mid(@@version, 1, 1)='5' and sleep(2);
    +---------------------------------------+
    | mid(@@version, 1, 1)='5' and sleep(2) |
    +---------------------------------------+
    |                                     0 |
    +---------------------------------------+
    1 row in set (2.00 sec)

    -- MSSQL
    select '' if(substring(@@version, 1, 1)='M') waitfor delay '0:0:5'; 

    -- SQLLITE
    -- LIKE('ABCDEFG',UPPER(HEX(RANDOMBLOB(300000000/2))))를 통해 시간 지연 발생 확인
    select case when substr(sqlite_version(), 1, 1)='3' then LIKE('ABCDEFG',UPPER(HEX(RANDOMBLOB(300000000/2)))) else 1=1 end;

  ```



## 시스템 테이블
### MySQL
  - information_schema,mysql,performance_schema,sys 데이터베이스가 존재함
  -  스키마,테이블 정보
    - TABLES 테이블을 통해 스키마의 정보를 조회할 수 있음. 아래를 살펴보면, TABLES 테이블의 내용 중 TABLE_SCHEMA에 해당하는 정보를 그룹화하여 조회
      ```sql
        -- MySQL 테이블 정보 조회
        mysql> select TABLE_SCHEMA, TABLE_NAME from information_schema.TABLES;
        /*
        +--------------------+----------------+
        | TABLE_SCHEMA       | TABLE_NAME     |
        +--------------------+----------------+
        | information_schema | CHARACTER_SETS |
        ...
        | DREAMHACK          | users          |
        | mysql              | db             |
        ...
        +--------------------+----------------+
        292 rows in set (0.01 sec)
        */
      ```
  - 컬럼 정보
    ```sql
      -- MySQL 컬럼 정보 조회
      mysql> select TABLE_SCHEMA, TABLE_NAME, COLUMN_NAME from information_schema.COLUMNS;
      /*
      +--------------------+----------------+--------------------+
      | TABLE_SCHEMA       | TABLE_NAME     | COLUMN_NAME        |
      +--------------------+----------------+--------------------+
      | information_schema | CHARACTER_SETS | CHARACTER_SET_NAME |
      ...
      | DREAMHACK          | users          | uid                |
      | DREAMHACK          | users          | upw                |
      ...
      | mysql              | db             | Db                 |
      | mysql              | db             | User               |
      ...
      +--------------------+----------------+--------------------+
      3132 rows in set (0.07 sec)
      */
    ```
  - 실시간 실행 쿼리 정보
    - PROCESSLIST 테이블을 통해 실시간으로 실행되는 쿼리를 조회. 아래를 살펴보면, PROCESSLIST 테이블의 내용을 모두 출력. 쿼리 결과에 조회하기 위해 입력한 쿼리가 포함
    ```sql
    -- MySQL 실시간 실행 쿼리 조회
    mysql> select * from information_schema.PROCESSLIST;
    /*
    +-------------------------------------------------+
    | info                                            |
    +-------------------------------------------------+
    | select info from information_schema.PROCESSLIST |
    +-------------------------------------------------+
    1 row in set (0.00 sec)
    */
    ```
  - SYS 데이터베이스의 SESSION 테이블을 통해 실행 중인 계정과 함께 조회
    ```sql
    MySQL 계정 및 실시간 실행 쿼리 조회
    mysql> select user,current_statement from sys.session;
    /*
    +----------------+------------------------------------------------+
    | user           | current_statement                              |
    +----------------+------------------------------------------------+
    | root@localhost | select user,current_statement from sys.session |
    +----------------+------------------------------------------------+
    1 row in set (0.05 sec)
    */
    ```
  - DBMS 계정 정보
   - USER_PRIVILEGES 테이블을 통해 MySQL 서버의 계정 정보를 조회
   - GRANTEE, PRIVILEGE_TYPE, 그리고 IS_GRANTABLE을 조회
     - PRIVILEGE_TYPE: 특정 사용자가 가진 권한의 종류
     - IS_GRANTABLE: 특정 사용자가 자신의 권한을 다른 유저에게 줄 수 있는지 YES 또는 NO로 표시
   ```sql
    MySQL DBMS 권한 및 계정 정보
    mysql> select GRANTEE,PRIVILEGE_TYPE,IS_GRANTABLE from information_schema.USER_PRIVILEGES;
    /*
    를 나타냅니다.
    됩니다.
    +-------------------------+-------------------------+--------------+
    | GRANTEE                 | PRIVILEGE_TYPE          | IS_GRANTABLE |
    +-------------------------+-------------------------+--------------+
    | 'root'@'localhost'      | SELECT                  | YES          |
    ...
    | 'root'@'localhost'      | SUPER                   | YES          |
    ...
    | 'user_test'@'localhost' | USAGE                   | NO           |
    +-------------------------+-------------------------+--------------+
    58 rows in set (0.00 sec)
    */ 
   ```
  - USER 테이블
    ```sql
    MySQL DBMS 계정 정보
    mysql> select User, authentication_string from mysql.user;
    
    -- authentication_string: 사용자의 비밀번호를 해싱한 값 입니다.
    +------------------+-------------------------------------------+
    | User             | authentication_string                     |
    +------------------+-------------------------------------------+
    | root             | *...                                      |
    | mysql.sys        | *THISISNOTAVALIDPASSWORDTHATCANBEUSEDHERE |
    | mysql.session    | *THISISNOTAVALIDPASSWORDTHATCANBEUSEDHERE |
    | user_test        | *...                                      |
    +------------------+-------------------------------------------+
    4 rows in set (0.00 sec)
    */
    ```
### MSSQL
- 데이터베이스 : master, tempdb, model, msdb
- 데이터베이스 정보 조회
  - master 데이터베이스의 SYSDATABASES 내 테이블 내용 중 name에 해당하는 정보를 조회
    ```sql
    SELECT name FROM master..sysdatabases; -- ..은 스키마를 생략할 때 사용. MSSQL에서 생략하면 기본으로 dbo 스키마 사용
    /*
    name
    -------
    master
    tempdb
    model
    msdb
    dreamhack # 이용자 정의 데이터베이스 (예시)
    */
    ```
  - DB_NAME(num) 형식으로 데이터베이스의 정보를 알 수 있음. 인자로 전달되는 숫자 중 0은 현재 데이터베이스를 의미
    - DB_NAME을 이용한 데이터베이스 정보 조회
      ```sql
      SELECT DB_NAME(1); -- master
      USE dreamhack; -- dreamhack DB 사용
      SELECT DB_NAME(0); -- 0은 현재 데이터베이스 이므로 dreamhack 출력됨
      ```
- 테이블 정보
  - SYSOBJECTS 테이블로 테이블 정보 조회
    - SYSOBJECTS 테이블의 내용 중 name에 해당하는 정보를 조회하되, xtype 이 'U'인 조건을 적용
    - xtype='U'는 이용자 정의 테이블을 의미합니다. 쿼리의 결과를 보면 이용자 정의 테이블인 "users"가 출력된 것을 확인가능
     ```sql
     SELECT name FROM dreamhack..sysobjects WHERE xtype = 'U'; -- 사용자 저의 테이블 출력 됨
     ```
  - 또는 INFORMATION_SCHEMA 스키마의 TABLES 테이블의 table_name을 조회
    ```sql
    SELECT table_name FROM dreamhack.information_schema.tables;
    ```
- 컬럼 정보
  - SYSCOLUMNS 테이블을 통해 컬럼의 정보를 조회
    ```sql
    SELECT name FROM syscolumns WHERE id = (SELECT id FROM sysobjects WHERE name = 'users');
    ```
  - 또는 INFORMATION_SCHEMA의 COLUMNS 테이블을 참조해 조회
    ```sql
    SELECT table_name, column_name FROM dreamhack.information_schema.columns;
    ```
- DBMS 계정 정보
  - master 데이터베이스의 sys.sql_logins 테이블을 통해 MSSQL 서버의 계정 정보를 조회 가능
  - sql_logins 테이블의 name , password_hash를 조회
    ```sql
    MSSQL 계정 정보 조회
    SELECT name, password_hash FROM master.sys.sql_logins;
    /*
    name		password_hash
    --------------------------
    sa			NULL
    dreamhack	NULL
    */

    -- master 데이터베이스의 syslogins 테이블을 통해 조회하는 방법도 가능
    SELECT * FROM master..syslogins;
    ```
