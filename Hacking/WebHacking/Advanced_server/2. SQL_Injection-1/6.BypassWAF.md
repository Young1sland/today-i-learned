### 연산자 검사 미흡
- and,or 필터링 우회
  ```sql
  select 1 || 1; -- or
  ```
- REGEXP: 정규 표현식 매칭. 정규 표현식을 사용하여 문자열 패턴을 비교
  ```sql
  SELECT * FROM users WHERE email REGEXP '^[A-Za-z0-9._%+-]+@[A-Za-z0-9.-]+\.[A-Za-z]{2,4}$';
  ```
- XOR: 배타적 논리합,두 조건 중 하나만 참일 때 참
  ```sql
  SELECT * FROM table_name WHERE condition1 XOR condition2;
  ```
- MATCH: 전문 검색. MATCH(column) AGAINST(expression) 형태로 사용. AGAINST와 함께 사용하여 특정 텍스트가 테이블의 컬럼에 포함되어 있는지 검색
  ```sql
  SELECT * FROM articles WHERE MATCH(title, content) AGAINST('database');
  ```
- DIV: IV는 정수 나눗셈을 수행, 정수 반환
  ```sql
  SELECT 7 DIV 2 FROM dual; --3 반환
  ```

### 공백 탐지
- 주석을 이용한 공백 문자 검사 우회
  ```sql
  SELECT/**/'abc';
  ```
- Backtick을 이용한 공백 문자 검사 우회
  ```sql
  select`username`,(password)from`users`WHERE`username`='admin';
  ```

### MySQL 우회기법
- 문자열 검사 우회
 ```sql
 select 0x6162, 0b110000101100010; --ab

 select char(0x61, 0x62); --ab

 select concat(char(0x61), char(0x62));  --ab

 select mid(@@version,12,1); -- 가젯 이용한 문자열 우회..?

 ```
- 공백 검사 우회
```sql
-- 개행 이용한 우회
> select
-> 1;

-- 주석 이용한 우회
> select/**/1;
```
- Mysql 주석 구문 실행
  - /**/는 주석으로 쿼리 구문으로 해석하지 않으나 MySQL /*!*/는 구문 분석 후 쿼리의 일부로 실행
 ```sql
 select 1 /*!union*/ select 2;

 SELECT * FROM table1 /*! UNION SELECT * FROM table2 */;

 -- MySQL 5.7.8 이상에서 실행될 때만 실행
 /*!50708 ALTER TABLE example ADD COLUMN new_column VARCHAR(255); */

 ```

 ### PostgreSQL 우회기법
 - 문자열 검사 우회
 ```sql
 postgres=> select chr(65) -- A
 postgres=> select concat(chr(65), chr(66)); -- AB
 postgres=> select substring(version(),23,1);  --가젯 이용한 검사 우회
 ```
- 공백 검사 우회
 ```sql
 -- 개행 이용
 postgres=> select
   1;

 -- 주석 이용
 postgres=> select/**/1;
 
 ```

### SQLite 우회
- 문자열 검사 우회
```sql
 sqlite> select char(0x61); -- a
 sqlite> select char(0x61)||char(0x62); --ab
 ```
- 공백 검사 우회
  ```sql
    -- 개행 이용
    sqlite> select
    ...> 1;

    -- 주석 이용
    sqlite> select/**/1; 
  ```
- 구문 검사 우회
  - select 사용하지 못할 때 `union values(num)` 이용
  ```sql
  sqlite> select 1 union values(2);
  /*
  1
  2
  */
  ```
