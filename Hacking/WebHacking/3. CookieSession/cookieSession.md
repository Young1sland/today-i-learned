# Cookie & Session

## HTTP 프로토콜 특징
- Connectionless  
  하나의 요청에 하나의 응답을 한 후 연결을 종료 함.  
  새 요청이 있을 때마다 항상 새로운 연결을 맺음.  
- Stateless  
  통신이 끝난 후 상태 정보를 저장하지 않는 것을 의미 함.  
  이전 연결에서 사용한 데이터를 다른 연결에서 요구할 수 없음.  

## Cookie
- HTTP에서 상태를 유지하기 위해 쿠키를 사용. 일반적으로 클라이언트 정보 기록, 상태 정보 표현.  
- 쿠키는 key와 value로 이뤄짐.  
- 서버가 클라이언트에 쿠키를 발급하면 클라이언트는 서버에 요청을 보낼 때마다 쿠키를 같이 전송.  
- 서버는 클라이언트의 요청에 포함된 쿠키를 확인해 클라이언트를 구분할 수 있음.  
- 쿠키는 통신할 때마다 전송되기 때문에 리소스 낭비가 발생할 수 있음.  
  최근에는 Modern Storage APIs를 토해 데이터를 저장하는 방식 권장.
## 세션
- 클라이언트가 쿠키를 변조해 서버에 요청을 보낼 수 있음. 클라이언트가 인증 정보를 변조할 수 없게 하기 위해 세션을 사용함.
- 인증 정보를 서버에 저장하고 해당 데이터에 접근할 수 있는 키(랜덤문자열)를 만들어 클라이언트에 전달하는 방식으로 작동 함. 이를 Session ID라고 함.
- 브라우저는 Session ID를 쿠키에 저장하고 이후에 HTTP 요청을 보낼 때 사용.  
  서버는 요청에 포함된 키에 해당하는 데이터를 가져와 인증 상태를 확인.
- 쿠키는 데이터 자체를 이용자가 저장하며 세션은 서버가 저장한다는 차이가 있음.

## 쿠키 적용법
- 쿠키는 클라이언트에 저장되기 때문에 클라이어느는 저장된 쿠키를 조회,수정,추가 가능
- 쿠키를 설정할 때 만료 시간 지정할 수 있음. 쿠키 만료는 클라이언트(브라우저)에서 관리
- 서버에서 쿠키 설정  
  HTTP 응답 헤더에 쿠키 설정 헤더(Set-Cookie) 추가하면 클라이언트 브라우저가 설정
  ```http
  HTTP/1.1 200 OK
  Server: Apache/2.4.29 (Ubuntu)
  Set-Cookie: name=test;
  Set-Cookie: age=30; Expires=Fri, 30 Sep 2022 14:54:50 GMT;
  ```
- 클라이언트에서 쿠키 설정
  자바스크립트를 사용해 쿠키 설정
  ```js
  document.cookie = "name=test;"
  document.cookie = "age=30; Expires=Fri, 30 Sep 2022 14:54:50 GMT;"
  ```
- 크롬에서 Cookie 확인
  - console에서 document.cookie 입력
  - Application Tab에서 확인
- 세션 하이재킹(Session Hijacking)  
  공격자가 이용자의 쿠키를 훔쳐 세션에 해당하는 이용자의 인증 상태를 획득.
