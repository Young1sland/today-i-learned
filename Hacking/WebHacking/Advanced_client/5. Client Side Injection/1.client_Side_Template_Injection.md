### Vue Template Injetion

```js
<script src="https://unpkg.com/vue@3"></script>
<div id="app">{{ message }}</div>
<script>
  Vue.createApp({
    data() {
      return {
        message: 'Hello Vue!'
      }
    }
  }).mount('#app')
</script>
```
`{{}}`로 감싸진 Vue 템플릿 부분 내에서 문자열을 표시하거나 JS 표현식을 실행할 수 있음.    
템플릿 내부에 공격자의 입력이 들어가 Template Injection 취약점이 발생하면 XSS 공격으로 이어질 수 있음.    


```js
<script src="https://unpkg.com/vue@3"></script>
<div id="app">
	<?php echo htmlspecialchars($_GET['msg']); ?>
</div>
<script>
  Vue.createApp({
    data() {
      return {
        message: 'Hello Vue!'
      }
    }
  }).mount('#app')
</script>
```
- htmlspecialchars : 문자열에서 특정한 특수 문자를 HTML 엔티티로 변환한다. 이함수를 사용하면 악성 사용자로 부터 XSS 공격을 방지 할 수 있다.
- htmlspecialchars 함수를 통해 꺽쇠 문자가 모두 인코딩되기 때문에 일반적인 XSS 공격은 불가능 하지만 출력되는 값이 Vue의 템플릿으로 사용될 수 있기 때문에 Template Injection이 발생한다.
- Vue Template Injection이 발생하는지 확인하는 간단한 방법은 템플릿을 이용해 산술 연산을 수행해 보는 것.    
  {{1+1}}입력 시 2가 출력되면 발생
- Template Injection이 발생할 때 이를 임의 자바스크립트 코드 실행으로 연계하는 방법으로 보통 생성자 이용.
  Vue템플릿 컨택스트 내에서 생성자에 접근하여 임의 코드에 해당하는 함수 생성하고 이를 호출하는 방식으로 XSS 공격 가능
  ```{{_Vue.h.constructor("alert(1)")()}}```과 같은 방법으로 공격 가능

### AngularJS
```js
<!doctype html>
<html ng-app>
  <head>
    <script src="https://ajax.googleapis.com/ajax/libs/angularjs/1.8.2/angular.min.js"></script>
  </head>
  <body>
    <div>
      <label>Name:</label>
      <input type="text" ng-model="yourName" placeholder="Enter a name here">
      <hr>
      <h1>Hello {{yourName}}!</h1>
    </div>
  </body>
</html>
```
`{{}}`로 감싸진 부분이 AngularJS 템플릿 부분이며, 공격자의 입력이 들어가 XSS 공격으로 이어질 수 있음.

```js
<!doctype html>
<html ng-app>
  <head>
    <script src="https://ajax.googleapis.com/ajax/libs/angularjs/1.8.2/angular.min.js"></script>
  </head>
  <body>
      <?php echo htmlspecialchars($_GET['msg']); ?>
  </body>
</html>
```
- 출력되는 값이 AngularJS의 템플릿으로 사용될 수 있기 때문에 Template Injection이 발생 함.
- 생성자에 접근하여 임의 ㅗ드에 해당하는 함수 생성하고 공격 가능
  `{{constructor.constructor("alert(1)")()}}`
