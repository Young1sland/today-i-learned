## Assembly
### 구조
|opcode|operand1|operand2|
|--|--|--|
|mov|eax,|3|

### 명령어
|||
|--|--|
|데이터 이동(Data Transfer)|mov, lea|
|산술 연산(Arithmetic)|inc, dec, add, sub|
|논리 연산(Logical)|and, or, xor, not|
|비교(Comparison)|cmp, test|
|분기(Branch)|jmp, je, jg|
|스택(Stack)|push, pop|
|프로시져(Procedure)|call, ret, leave|
|시스템 콜(System call)|syscall|

### 피연산자
- 상수(Immediate Value), 레지스터(Register), 메모리(Memory) 3종류가 올 수 있음.
- 메모리 피연산자는 []으로 둘러싸인 것으로 표현 됨. 앞에 TYPE PTR이 추가될 수 있음.    
  TYPE : BYTE, WORD, DWORD, QWORD (1,2,4,8 바이트) 
  ```QWORD PTR [0x8048000] 0x8048000의 데이터를 8바이트만큼 참조```

### x86-64 어셈블리 명령어
1. 데이터 이동    
  어떤 값을 레지스터나 메모리에 옮기도록 지시    
  
    |mov dst,src|src에 들어있는 값을 dst에 대입|
    |--|--|
    |mov rdi,rsi|rsi의 레지스터 값을 직접 rdi에 대입|
    |mov rdi,[rsi]|rsi의 메모리 값을 rdi에 대입|
    |mov QWORD PTR[rdi+8*rcx], rsi|rsi의 값을 rdi+8*rcx가 가리키는 주소에 대입|    

    |lea dst,src|src의 유효 주소를 dst에 대입|
    |--|--|
    |lea rsi, [rbx+8*rcs]|rdi+8*rcx를 rsi에 대입|

    ```
    [Register]
    rbx = 0x401A40
    =================================
    [Memory]
    0x401a40 | 0x0000000012345678
    0x401a48 | 0x0000000000C0FFEE
    0x401a50 | 0x00000000DEADBEEF
    0x401a58 | 0x00000000CAFEBABE
    0x401a60 | 0x0000000087654321
    =================================
    [Code]
    1: mov rax, [rbx+8] // rbx+8 메모리 값을 rax에 넣음. 실행하면  rax = 0xC0FFEE
    2: lea rax, [rbx+8] // 실행하면 rax = 0x401A48
    ```

4. 비교    
   - cmp op1,op2    
     두 피연산자를 빼서 비교. 연산 결과는 op1에 대입X.  두수가 같으면 ZF 플래그 설정 됨
        ```
       
        [Code]
        mov rax, 0xA
        mov rbx, 0xA
        cmp rax, rbx

        실행하면 ZF=1로 설정 됨
        ```

   - test op1, op2    
     두 피연산자를 AND 비트연산. 연산 결과는 op1에 대입X. AND 결과가 0이면 ZF 플래그 설정 됨
        ```
        xor rax,rax
        test rax,rax

        실행하면 ZF=1        
        ```    
5. 분기    
   rip를 이동시켜 실행 흐름 바꾼다.
   - jmp addr: addr로 rip를 이동시킵니다.
        ```
        [Code]
        1: xor rax, rax
        2: jmp 1 ; jump to 1        
        ```
    - je addr: 직전에 비교한 두 피연산자가 같으면 점프 (jump if equal)
        ```
        [Code]
        1: mov rax, 0xcafebabe
        2: mov rbx, 0xcafebabe
        3: cmp rax, rbx ; rax == rbx
        4: je 1 ; jump to 1
        ```
    - jg addr: 직전에 비교한 두 연산자 중 전자가 더 크면 점프 (jump if greater)
        ```
        [Code]
        1: mov rax, 0x31337
        2: mov rbx, 0x13337
        3: cmp rax, rbx ; rax > rbx
        4: jg 1  ; jump to 1
        ```

    - 연산

      [rsp]=val
      ```
    - 예제
      ```

      rsp = 0x7fffffffc400

      0x7fffffffc400 | 0x0  <= rsp

      ```
      결과
      ```
      [Register]
      rsp = 0x7fffffffc3f8
      [Stack]

      0x7fffffffc408 | 0x0
      ```
  - pop reg : 스택 최상단의 값을 꺼내서 reg에 대입

### 잠깐 정리
- rsi(source index) : 데이터를 옮길 때 원본을 가리키는 포인터
- rax(accumulator register) : 함수의 반환 값
- rbp(stack base pointer) : 스택의 바닥을 가리키는 포인터
  - 함수가 호출될 때, 이전 함수의 rbp 값이 스택에 저장되고, 현재 rsp 값이 rbp에 복사 됨. 이렇게 하여 현재 함수의 스택 프레임이 시작되는 지점을 기록
  - 함수가 종료될 때, 스택에서 팝(pop) 연산을 수행하여 이전 함수의 rbp 값을 복원. 이는 함수 호출 시점으로 스택 포인터를 되돌리는 역할을 함
- rsp(stack pointer) : 사용 중인 스택의 위치를 가리키는 포인터
- rip : 명령어 레지스터

### 스택 프레임 할당과 해제
1. call func 호출, 이 때 다음 명령어 주소를 스택에 push
2. jmp address 후 기존의 스택 프레임 저장 위해 rbp를 스택에 push
   - push rbp
3. 새로운 스택 프레임 만들기 위해 rbp를 rsp로 옮김
   - mov rbp, rsp
4. 새로 만든 스택 프레임 공간 확장 위해 rsp를 0x30만큼 뺀다.
   - sub rsp, 0x30
5. 할당한 스택 프레임에 지역변수를 할당한다.
   - mov BYTE PTR[rsp], 0x3
6. 스택 프레임 위에서 여러 연산 수행
7. 저장해뒀던 rbp를 꺼내서 원래의 스택프레임으로 돌아간다.
   - leave
      - move rsp, rbp
      - pop rbp
8. 저장해뒀던 반환 주소를 꺼내어 원래의 실행 흐름으로 돌아간다.
   - ret
      - pop rip
### 예제
```
[Code]
main:
    push rbp
    mov rbp, rsp
    mov esi, 0xf
    mov rdi, 0x400500
    call 0x400497 <write_n>
    mov eax, 0x0
    pop rbp
    ret
    
write_n:
    push rbp
    mov rbp, rsp
    mov QWORD PTR [rbp-0x8],rdi
    mov DWORD PTR [rbp-0xc],esi
    xor rdx, rdx
    mov edx, DWORD PTR [rbp-0xc]
    mov rsi,QWORD PTR [rbp-0x8]
    mov rdi, 0x1
    mov rax, 0x1
    syscall
    pop rbp
    ret
    
==================================
